# 个人任务管理范式

> 基于 Obsidian + TaskNotes 4.5.3 的个人工作任务管理规则，2026-05-12 立。
> 本文档为说明文档，不参与 TaskNotes 任务查询（无 `task` 标签）。

## 一、任务粒度

**1 任务 ≈ 1 小时**（小时粒度）

- 不超过 2 小时的工作单元 → 1 个任务
- 超过 2 小时的工作 → 按可交付物或子步骤拆为多个任务（如"需求文档"和"原型"分开）
- 用 `timeEstimate` 字段标注预估工时（分钟）

## 二、状态流转

仅使用 TaskNotes 配置的三个原生状态：

| 状态 | 含义 | 进入时机 |
|---|---|---|
| open | 未开始 | 任务创建后默认 |
| in-progress | 进行中 | 当日要推进的任务，开工时切换 |
| done | 已完成 | 任务交付物已产出 |

**阻塞处理**：不使用独立的 blocked 状态，改用 `contexts` 字段标记 `@阻塞`，并在任务正文中说明等待对象（人/事项），状态保持 in-progress。

## 三、字段约定

每个任务必填或推荐填写的 frontmatter 字段：

| 字段 | 必填 | 取值规则 |
|---|---|---|
| `title` | 是 | 动词+对象，如"调整通用人群原型 v0.3" |
| `status` | 是 | open / in-progress / done |
| `priority` | 是 | none / low / normal / high（high = 当日重点） |
| `scheduled` | 是 | 计划执行日期 YYYY-MM-DD |
| `due` | 否 | 截止日期（评审节点、迭代终止日） |
| `projects` | 推荐 | 产品模块，如 `+个人数字空间` `+健康画像` |
| `contexts` | 推荐 | 工作场景，如 `@设计` `@评审` `@开发跟进` `@外部会议` `@阻塞` |
| `timeEstimate` | 推荐 | 预估工时（分钟） |
| `tags` | 是 | 必含 `task` 标签（否则 TaskNotes 不识别） |

**正文必含**：用 Obsidian wikilink 关联相关需求/原型文档，如：

```markdown
关联文档：
- [[个人数字空间_PC端融合通用人群健康空间_需求文档_v0.3_20260511]]
- [[个人数字空间_PC端融合通用人群健康空间_原型_v0.2_20260508]]
```

## 四、优先级判定

沿用全局 CLAUDE.md 决策风格：

1. **high**：合规阻塞 / 评审节点驱动（如评审会前必须完成）
2. **normal**：业务价值高、本期主线
3. **low**：用户体验优化、可延后
4. **none**：探索性、未排期

## 五、工作节奏

### 每日

- **早晨**（9:00 配合 Cron 提醒）：查看 Agenda 视图（`scheduled=today`），确认今日重点
- **傍晚**（下班前 15min）：更新 Kanban 视图，把已完成任务拖入 done 列；未完成任务判断是否顺延 scheduled

### 每周

- **周一**：在 plan 文件中起草本周计划，把任务批量录入 TaskNotes，scheduled 分布到周一至周五
- **周五**：Kanban 视图按 status 分组，截图或文字记录本周完成情况，归档进入 plan 文件

### 每迭代（双周）

- **迭代末**：导出本迭代所有 done 任务清单为 markdown，作为附录追加到项目记忆 session 文件（`session_YYYYMMDD_迭代回顾.md`）
- 用于：产品交付凭证、回顾本迭代节奏

## 六、文件命名

任务文件名格式（TaskNotes 配置 `storeTitleInFilename: true`）：

```
{时间戳}-{标题}.md
```

实际由 TaskNotes 自动生成 zettel 风格的文件名，无需手动管理。

## 七、视图说明

| 视图 | 用途 | 何时查看 |
|---|---|---|
| Agenda | 按日期看任务 | 每日早晨 |
| Kanban | 按状态看流转 | 每日傍晚、周五 |
| Tasks | 列表全览 | 周一规划、迭代末导出 |
| Calendar | 月历视图 | 评审节点排期 |
| Mini Calendar | 紧凑日历 | 侧边栏常驻 |
| Relationships | 任务关系图 | 大任务拆分时 |

## 八、与 Cron 提醒的协作

每工作日 9:00 自动触发的 Claude 助理会：
1. 第 1 步：读取昨日 TaskNotes 任务状态，分组报告
2. 第 2 步：等用户点评后，引导制定今日计划并按本范式录入新任务

详见：[[fancy-purring-quasar]] 任务 5 段落。
