**负责人**：产品经理

## 核心业务场景

1. **医生端** - AI诊前摘要与画像：问题导向型健康画像，推理证据链，临床洞察
2. **居民端** - AI居民健康管家：通俗易懂的健康指南，有温度的健康服务
3. **监管端** - AI决策秘书：群体健康画像，疾病图谱分析，政策决策支撑

## 文档命名规范

**格式**：`{模块名称}_{文档类型}_{版本号}_{日期}.{扩展名}`

**示例**：
- `人群健康空间_需求文档_v1.0_20260403.md`
- `人群健康空间_数据模型_v1.1_20260401.docx`
- `人群健康空间_原型设计_v1.1_20260410.html`

**版本标识**：
- 正式版本：v1.0、v1.1、v2.0
- 草稿版本：文件名加"_草稿"标识
- 旧版本：移至"归档"目录，文件名加"_旧版"标识

---

## 维护规则

1. **新增文档**：按命名规范创建，同步更新对应目录的README索引
2. **版本更新**：保留最新版本和上一个正式版本，其余移至归档
3. **文档归档**：超过2个版本的旧文档移至"归档/YYYY-MM/"目录
4. **定期清理**：每周检查临时文件，每月回顾目录结构
