# PC端业务后台原型风格

## 概述

使用这个 skill 生成 PC 端业务后台低保真 HTML 原型，重点表达信息架构、模块分组、筛选表格、数据看板、详情面板和操作路径。默认语境是医疗健康、居民健康画像、个人数字健康空间、用户中心、数据管理平台等业务后台。

这不是高保真视觉稿 skill。输出应接近可评审的后台线框原型：结构清晰、蓝色状态强调克制、浅蓝灰背景、白色卡片、表格和筛选稳定、医疗健康文案自然。

如需了解截图来源和更细风格证据，按需读取 `references/style-reference.md`。

## 工作流程

1. 先判断页面类型：门户卡片页、左侧菜单列表页、个人数字空间看板、标签图谱页、详情页、表单页、弹窗/抽屉页或混合后台页面。
2. 明确页面主框架，只保留一种主导航体系：顶部模块导航、左侧菜单导航或顶部患者信息条，不要同时堆叠多套主导航。
3. 将用户需求转译为后台产品文案和模块标题，不要直接把需求描述贴到页面上。
4. 默认生成单文件 HTML，内联 CSS；除非用户明确要求图片、React、Vue 或其他格式。
5. 使用低保真表达：真实业务结构优先，视觉装饰只保留能帮助识别平台语境的浅蓝、卡片、标签、空状态和线框占位。
6. 交付前自检：筛选项是否对齐、表格列是否可读、操作列是否换行有序、卡片是否等宽、文字是否不遮挡、分页是否落在表格底部。

## 布局规则

### 画布与框架

- 使用 PC 宽屏画布，默认面向 `1440px` 到 `1920px` 宽度。
- 页面背景优先使用浅蓝灰或极浅灰，内容区使用白色卡片。
- 常见框架可选：
  - 顶部蓝色品牌栏 + 横向模块导航 + 内容卡片门户。
  - 顶部蓝色品牌栏 + 左侧浅色菜单 + 右侧筛选表格。
  - 居中内容容器 + 患者/居民信息条 + 看板卡片 + 画像/图谱区域。
  - 顶部患者信息条 + 标签统计卡 + 大面积图谱画布。
- 内容最大宽度可在 `1280px` 到 `1440px` 之间；列表管理页可铺满右侧内容区。
- 页面纵向节奏默认使用 `16px`、`20px`、`24px`，模块内使用 `8px`、`12px`。

### 顶部栏

- 顶部品牌栏常用蓝色，左侧为系统名称，右侧为用户身份、头像/图标和轻量操作。
- 横向模块导航位于品牌栏下方或内容顶部，当前项用蓝色文字、下划线或浅蓝底表示。
- 顶部患者信息条应轻量、扁平，展示姓名脱敏、性别、年龄、状态、签约机构/医生、数据截止时间等核心上下文。
- 顶部栏高度要克制，不要做成大 hero 或营销页头。

### 左侧菜单

- 左侧菜单适合数据管理、居民管理、签约管理、系统使用情况等后台模块。
- 菜单宽度默认 `180px` 到 `220px`，背景白色或极浅蓝灰。
- 一级菜单使用图标占位 + 文案 + 展开箭头；当前二级菜单使用浅蓝底、蓝色文字和小蓝点。
- 左侧底部可放淡化的业务插画占位，但不得抢占主任务视线。

### 内容区

- 门户页使用分组标题 + 卡片网格，常见为 3 到 4 列，卡片等宽等高。
- 列表页使用筛选卡 + 表格卡；筛选卡在上，表格标题和表头紧随其后。
- 看板页使用统计卡、重点关注卡、记录卡、人体/画像占位、空状态卡组合。
- 图谱页使用指标概览卡 + 大画布区域 + 图例/筛选浮层。
- 详情页使用分组信息卡，字段采用左 label 右 value 的稳定两列或多列布局。

## 组件规则

### 卡片

- 常规卡片使用白底、细边框或轻阴影，圆角默认 `8px`。
- 卡片内标题加粗，说明文字使用灰色；图标或图片位作为右侧占位，不要越界。
- 门户入口卡内通常包含：标题、简短说明、右侧图标占位。
- 统计卡通常包含：指标名、数字、解释文字、右侧小图标或状态点。

### 筛选与表单

- 筛选区使用横向网格排布，label 固定宽度，输入框和选择框等高。
- 常见筛选项：居民姓名、证件号码、性别、年龄范围、签约机构、签约医生、人群分类、日期范围。
- 主按钮使用蓝色填充，次按钮使用白底蓝色描边或浅色描边。
- 表单页字段要分组，不要把大量字段无间隔堆成单列。

### 表格

- 表格适合居民查询、画像查询、审核记录、检查/检验/用药记录。
- 表头使用浅蓝灰底，正文白底，行分割线浅灰。
- 操作列放在最右侧，可多行排列链接；主要操作蓝色，禁用或次要操作灰色。
- 身份证、手机号等敏感信息默认脱敏。
- 状态可使用小标签表达，如未签约、门诊、住院、高风险、中风险、低风险。
- 分页固定在表格底部右侧，包含总数、每页条数、页码和跳转输入。

### 看板与图谱

- 看板页可使用蓝色渐变大区域，但保持低保真，不做复杂纹理。
- 空状态使用浅蓝图标占位 + `暂无数据`，避免堆装饰。
- 人体图像、数字人、健康画像可用半透明人体占位或线框占位表达。
- 标签图谱用节点、虚线关系和图例表达；节点颜色区分基础信息、健康空间、高风险等类别。

### 弹窗与抽屉

- 弹窗用于审核、确认、编辑少量字段；抽屉用于查看详情和多分组信息。
- 标题区、内容区、底部操作区要分明。
- 默认按钮顺序：取消在左，确认/提交在右。

## 文案规则

- 使用医疗健康后台语境：居民健康画像、健康图像管理、签约居民管理、个人数字空间、标签图谱、重点关注、检查记录、检验记录、用药记录。
- 用户描述的是功能范围时，要转译成自然模块名，例如 `分类商品展示` 不应原样出现，后台可改成 `服务目录`、`记录列表`、`数据看板`。
- 状态文案要明确可操作或可解释，例如 `暂无说明`、`暂无数据`、`数据截止 2026-05-11`、`需要重点关注`。
- 不使用营销话术、夸张口号和移动端 App 式文案。

## 输出规则

- 默认输出独立 HTML 文件，内联 CSS，结构分区清晰，类名可读。
- 原型必须能直接用浏览器打开预览，不依赖远程资源。
- 可以使用简单 Unicode 符号或 CSS 形状作为图标占位，不强依赖图标库。
- 不输出后台代码、接口实现或数据库结构，除非用户明确要求。
- 如果用户提供新的 PC 截图，先抽取稳定规则，再生成页面；不要逐像素复刻。

## 自检清单

- 页面是否明显是 PC 业务后台，而不是移动端页面或官网。
- 是否保留蓝色医疗健康后台气质，且没有高保真过度装饰。
- 顶部栏、侧边栏、内容区之间是否边界清楚。
- 筛选项、表头、表格列、操作列、分页是否对齐。
- 卡片网格是否等宽，标题和说明是否不溢出。
- 图谱或人体画像区域是否有足够画布空间，不与控制项重叠。
