# PC端医疗健康业务后台线框原型风格参考

## 说明

这份参考文档用于沉淀截图组中的 PC 端后台风格来源和绘制标准。实际使用 skill 时，优先遵循 `SKILL.md` 的核心规则；当用户提供新的同类截图时，可继续把稳定特征追加到本参考中。

当前截图样式来自医疗健康后台和个人数字健康空间，包含：用户中心门户、居民健康画像管理列表、个人数字空间看板、个人健康标签图谱。

## 核心风格标记

- PC 宽屏后台，常见视口约 `1920x900`。
- 主色为医疗蓝，顶部品牌栏或模块头部使用蓝色。
- 页面底色常用浅蓝灰，内容容器为白色卡片。
- 字号整体克制：正文约 `14px`，标题约 `16px` 到 `22px`，数字指标可更大。
- 卡片圆角多为 `8px` 到 `10px`，低保真生成时默认统一 `8px`。
- 模块间距通常为 `16px`、`20px`、`24px`，卡片内间距为 `12px` 到 `20px`。
- 蓝色只承担品牌、选中态、主按钮、链接、状态强调，不要把页面做成大面积深蓝。
- 可出现浅蓝渐变块，但只用于看板背景或提示条，不应变成营销 hero。

## 截图中的页面类型

### 用户中心门户

- 顶部深蓝品牌栏，左侧系统品牌，右侧管理员身份。
- 第二层横向导航，当前模块以蓝色下划线突出。
- 主内容居中，分组标题下放入口卡片网格。
- 入口卡片为白底、轻阴影、右侧图标占位，常见四列布局。
- 背景可有淡蓝业务插画，但必须弱化。

### 居民健康画像管理列表

- 顶部蓝色系统栏，左侧白色菜单导航，右侧主内容。
- 内容顶部是标签页或当前位置条。
- 筛选区使用白色卡片，字段横向排布，两行以内优先。
- 表格标题明确，例如 `居民健康画像查询`。
- 表头浅蓝灰底，行高较宽松，右侧操作列多链接换行排列。
- 分页在底部右侧，含总数、每页条数、页码、跳转。

### 个人数字健康空间看板

- 页面居中容器，顶部患者信息条展示姓名脱敏、性别、年龄、签约状态等。
- 使用 `AI智能推荐`、科室/专科切换等轻量控制。
- 主看板使用蓝色渐变容器，内部包含重点关注、既往史、过敏史、近期异常指标、近期体征等卡片。
- 右侧可放人体/数字人占位，左侧或内侧有器官/系统列表。
- 下方有检查记录、检验记录、用药记录等白色记录卡。

### 个人健康标签图谱

- 顶部患者信息条和数据截止时间。
- 上方统计卡展示标签类别、命中标签数、高风险标签数。
- 主体为大面积白色图谱画布。
- 左侧可浮动图例和可见性控制。
- 图谱节点使用浅蓝、绿色、橙红等小面积状态色，关系线为浅灰虚线。

## 组件细节

- 主按钮：蓝底白字，常见于搜索、确认、提交。
- 次按钮：白底蓝字描边，常见于重置、取消。
- 链接操作：蓝色文字，禁用态或次要项灰色。
- 标签：小圆角或胶囊，颜色表达签约状态、风险等级、记录类型。
- 输入框：白底、浅灰边框、`32px` 到 `36px` 高。
- Select：右侧有箭头，支持 `全部`、机构、医生、人群分类等选项。
- 空状态：浅蓝图标或线框占位 + `暂无数据`。
- 医疗业务图标：可用简化蓝色图标占位，不要求真实图标细节。

## 复用建议

- 当需求是“某某管理系统首页”时，优先使用门户卡片页。
- 当需求是“查询、管理、审核、列表”时，优先使用左侧菜单 + 筛选区 + 表格。
- 当需求是“个人空间、健康画像、详情总览”时，优先使用顶部患者信息条 + 看板卡片。
- 当需求是“标签、关系、图谱、画像关系”时，优先使用统计卡 + 大画布图谱。
- 后续如果提供更多截图，应补充具体证据，但不要把单张截图中的偶然缺陷写成规则。
