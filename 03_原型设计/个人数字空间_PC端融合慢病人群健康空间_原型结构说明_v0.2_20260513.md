1. **个人数字空间主界面**
2. **慢病健康空间详情页**
3. **其他专项人群占位页**

## 四、页面1：个人数字空间主界面

### 4.1 页面定位

作为患者个体画像总入口，继续承载：

- 个人基础信息查看
- 标签能力查看
- 人群归属查看
- 进入人群健康空间详情页

### 4.2 页面结构

页面按纵向分为 3 个主区域：

```text
┌────────────────────────────────────────────────────────────┐
│ A. 顶部个人信息区                                           │
├────────────────────────────────────────────────────────────┤
│ B. 标签及人群统计卡区                                       │
├────────────────────────────────────────────────────────────┤
│ C. 个人健康标签图谱区                                       │
└────────────────────────────────────────────────────────────┘
```

### 4.3 本期入口变更

主界面整体不改版，仅调整“人群健康空间”卡片中的慢病入口：

| 入口 | 原行为 | 本期行为 |
|------|--------|----------|
| 通用人群 | 进入通用人群详情页 | 本慢病原型中保留占位，不展开 |
| 慢病 | 进入专项人群占位页 | 进入慢病健康空间详情页 |
| 孕产妇/重疾 | 进入专项人群占位页 | 保持占位页 |

### 4.4 C区：个人健康标签图谱区

保留当前能力，本期不做结构改造。原型中仅保留低保真节点和图例，作为个人数字空间主界面的上下文。

## 五、页面2：慢病健康空间详情页

### 5.1 页面定位

承载当前患者在“慢病健康空间”下的核心慢病管理数据。目标用户为健康管理人员，重点支持：

- 查看慢病诊疗阶段演进
- 查看血压、血糖、血脂等核心指标趋势
- 查看公卫慢病随访记录
- 查看高血压、糖尿病专病档案摘要

### 5.2 页面整体结构

采用：**顶部信息区 + 慢病病种卡 + 2×2 卡片网格**

```text
┌───────────────────────────────────────────────────────┐
│ A. 页面头部：返回入口 / 患者基础信息 / 空间标识          │
├───────────────────────────────────────────────────────┤
│ B. 慢病病种卡：高血压 / 糖尿病                          │
├───────────────────────────────────────────────────────┤
│ ┌─────────────────────┐  ┌─────────────────────┐     │
│ │ C1：慢病健康视图     │  │ C2：健康指标         │     │
│ │ 子菜单/搜索/列表     │  │ 子菜单/统计/趋势图   │     │
│ └─────────────────────┘  └─────────────────────┘     │
│ ┌─────────────────────┐  ┌─────────────────────┐     │
│ │ C3：随访记录         │  │ C4：专病档案         │     │
│ │ 表格/详情弹窗        │  │ 档案卡/待确认提示     │     │
│ └─────────────────────┘  └─────────────────────┘     │
└───────────────────────────────────────────────────────┘
```

### 5.3 布局要求

- 页面头部固定在顶部。
- 慢病病种卡位于头部内，用于承载高血压、糖尿病状态。
- 四模块以 2×2 网格卡片形式展示，占据剩余视口高度。
- 每个卡片内部支持独立滚动。
- 宽屏下 2×2 网格，窄屏降级为单列堆叠。

### 5.4 A区：页面头部

建议内容：

- 返回按钮 / 面包屑
- 页面标题：慢病健康空间
- 患者基础信息
- 数据截止时间

### 5.5 B区：慢病病种卡

展示患者命中的慢病类型：

- 高血压：展示管理状态、初诊时间、分型待确认提示
- 糖尿病：展示管理状态、初诊时间、复查提醒

说明：慢病分型映射依赖疾病知识库，原型中只表达字段位置，不固化诊断码规则。

## 六、C1：慢病健康视图模块

### 6.1 模块结构

```text
卡片标题：慢病健康视图
  ├─ 子菜单：前期风险 / 初诊确诊 / 药物管理 / 复发控制
  ├─ 搜索筛选区：关键词 + 病种 + 时间范围
  ├─ 列表或表格展示区
  └─ 详情弹窗
```

### 6.2 子菜单说明

| 子菜单 | 原型承载内容 | record 数量 |
|--------|--------------|-------------|
| 前期风险 | 展示首次确诊前的血压、血糖风险线索 | 0 |
| 初诊确诊 | 展示最早一条高血压/糖尿病诊断记录 | 2 |
| 药物管理 | 列表展示用药记录、来源、依从性 | 2 |
| 复发控制 | 展示疑似并发症和复发风险线索 | 1 |

### 6.3 交互要求

- 默认激活”前期风险”子菜单。
- 点击子菜单切换当前卡片内容，列表区域只显示对应分类的 record（通过 `data-type` 属性过滤）。
- 点击诊疗阶段事件打开详情弹窗。
- 药物管理使用列表展示（每条：药品名+剂量+频次），点击打开弹窗展示完整字段。
- 前期风险异常值可用强调色高亮，但阈值说明保持”待专家确认”边界。

### 6.4 技术实现

- 每条 record 添加 `data-type` 属性（risk/diag/med/relapse）
- `switchChronic()` 函数根据 type 参数控制 record 显隐：`r.style.display = r.getAttribute('data-type')===type ? '' : 'none'`

## 七、C2：健康指标模块

### 7.1 模块结构

```text
卡片标题：健康指标
  ├─ 子菜单：血压 / 血糖 / 血脂
  ├─ 统计卡区：历年、近一年、确诊后检查/异常次数
  ├─ 趋势图区：折线、数据点、目标线占位
  ├─ 缩放进度条占位
  └─ 指标溯源弹窗
```

### 7.2 原型表达重点

- 子菜单按病种动态展示：仅高血压→血压+血脂，仅糖尿病→血糖+血脂，双病种→三项全展示。
- 默认选中：仅高血压→血压，仅糖尿病→血糖，双病种→血压。
- 趋势图区展示目标线占位，文案明确“目标待专家确认”。
- 点击趋势图数据点打开指标溯源弹窗。

### 7.3 待确认边界

- 慢病人群血压、血糖、血脂控制目标未确认。
- 血脂展示项目未确认。
- 因此原型不固化任何判定阈值，只表达趋势、统计和溯源位置。

## 八、C3：随访记录模块

### 8.1 模块结构

```text
卡片标题：随访记录
  ├─ 年份快速跳转下拉框
  ├─ 宽屏表格
  └─ 随访详情弹窗
```

### 8.2 表格字段

原型表格展示以下字段：

- 随访时间
- 随访类型
- 随访方式
- 随访医生
- 本次血压
- 本次血糖
- 用药依从性
- 下次随访时间
- 操作

### 8.3 交互要求

- 按随访时间降序排列。
- 点击”查看详情”打开弹窗（880px 宽，复用 `.diag-modal` 样式类）。
- 弹窗内容分为五个区块：
  1. **随访详情**：随访医生、随访机构、本次随访日期、下次随访日期、随访类型标签
  2. **核心指标**：收缩压/舒张压蓝色渐变大卡（`background:linear-gradient(135deg,#3b82f6,#2563eb)`）+ 体重、BMI、心率小卡片（3 列网格）
  3. **生活方式目标**：吸烟、饮酒、运动频率、运动时长、摄盐情况，每行显示”当前值”和”目标值”对比（3 列网格：标签 90px / 当前值 / 目标值），浅灰背景行项
  4. **用药**：服药依从性、不良反应（2 列网格）+ 药品卡片列表（药品名、用法用量、服用时间）
  5. **医生健康指导**：一段指导文字，浅灰背景文本框
- 无随访记录时展示”暂无随访记录”。

## 九、C4：专病档案模块

### 9.1 模块结构

```text
卡片标题：专病档案
  ├─ 慢病专案摘要（高血压 + 糖尿病）
  ├─ 家医签约状态
  ├─ "查看完整档案"按钮
  └─ 完整档案弹窗（#archiveModal）
```

### 9.2 右下区域布局（摘要展示）

- **慢病专案摘要**：2 行卡片（高血压/糖尿病），每行包含：
  - 病种名称（带 emoji 图标）
  - 确诊时间
  - 当前阶段标签（蓝色胶囊，如"控制期"）
- **家医签约状态**：3 个字段（签约状态、签约医生、到期日期）
- **底部按钮**：`<button class="btn" onclick="openModal('archiveModal')">查看完整档案</button>`

### 9.3 完整档案弹窗（#archiveModal）

弹窗宽度 880px，复用 `.diag-modal` 样式类，内部可滚动，分为四个区块：

**1. 基本信息**（14 字段）
- 姓名、性别、出生日期、身份证号、手机号、民族、血型、婚姻状况、档案编号、建档日期、管辖医生、档案完整度（2 列网格）
- 现居地址、户籍地址（各占满一行）

**2. 慢病专案管理**
- 高血压：确诊时间、当前阶段（蓝色胶囊标签）、责任医生、管理机构
- 糖尿病：确诊时间、当前阶段（蓝色胶囊标签）、责任医生、管理机构
- 慢病专案标题用蓝色（`color:#2563eb`）

**3. 家医签约**（7 字段）
- 签约状态、签约医生、签约团队、签约机构、签约日期、到期日期（2 列网格）
- 服务包（占满一行）

**4. 过敏史 / 家族史**
- 过敏药物（红色高亮，`color:#dc2626;font-weight:600`）
- 父亲病史、母亲病史

### 9.4 空数据态

未建档时建议展示：

```text
暂无专病档案
当前患者已命中慢病标签，但 ADS 专病档案暂未归集。
```

本版 HTML 原型使用“已建档”示例态，未建档态可在设计稿中补充。

## 十一、关键交互说明

### 11.1 本次原型必须体现的交互

1. 主界面点击”慢病”进入慢病健康空间详情页。
2. 详情页显示头部、慢病病种卡、四模块 2×2 网格。
3. 慢病健康视图子菜单可切换，点击后只显示对应分类的 record。
4. 慢病健康视图点击诊疗记录打开诊疗详情弹窗，就诊详情区块包含 8 个字段：
   - 前 6 个字段：就诊时间、就诊机构、就诊科室、主诊医生、主诉、症状描述（2 列网格）
   - 诊断名称（占满一行）
   - 一般检查（占满一行）：展示血压和空腹血糖数据，异常值用红色文字（`style=”color:#dc2626”`）
5. 健康指标血压、血糖、血脂可切换。
6. 健康指标数据点点击后打开溯源弹窗。
7. 随访记录点击后打开详情弹窗（880px 宽，五个区块：随访详情/核心指标/生活方式目标/用药/医生健康指导）。
8. 专病档案点击”查看完整档案”按钮打开弹窗（880px 宽，四个区块：基本信息/慢病专案管理/家医签约/过敏史家族史）。
9. 返回个人数字空间路径可用。

### 11.2 可延后处理的交互

1. 极端数据量下分页、懒加载。
2. 趋势图区真实图表组件和缩放拖拽。
3. 药物记录、专病档案的深层详情页。
4. 慢病 AI 摘要与个性化建议模块。
5. 未建档居民的建档引导交互。

## 十二、验收关注点

- 主界面保持个人信息区、统计卡区、标签图谱区的结构。
- 慢病入口点击后进入慢病详情页，而非占位页。
- 慢病详情页包含慢病健康视图、健康指标、随访记录、专病档案四模块。
- 四模块以 2×2 卡片网格展示，每个卡片独立滚动。
- 页面中所有待确认医疗规则都有占位说明，不被误解为最终规则。
- 弹窗能打开和关闭，返回主界面路径可用。
