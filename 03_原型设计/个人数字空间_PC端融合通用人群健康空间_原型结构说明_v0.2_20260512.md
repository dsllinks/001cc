## 四、页面1：个人数字空间主界面

### 4.1 页面定位

作为患者个体画像总入口，继续承载：
- 个人基础信息查看
- 标签能力查看
- 人群归属查看
- 进入人群健康空间详情页

### 4.2 页面结构

页面按纵向分为 3 个主区域：

```text
┌────────────────────────────────────────────────────────────┐
│ A. 顶部个人信息区                                           │
├────────────────────────────────────────────────────────────┤
│ B. 标签及人群统计卡区                                       │
├────────────────────────────────────────────────────────────┤
│ C. 个人健康标签图谱区                                       │
└────────────────────────────────────────────────────────────┘
```

### 4.3 A区：顶部个人信息区

#### 作用
展示当前患者的基础身份信息与数据时间信息。

#### 建议字段
- 姓名、性别、年龄、民族
- 证件信息、联系方式、地址
- 数据截止时间、刷新按钮

#### 原型表现建议
- 采用横向信息条布局
- 左侧展示患者基础身份信息
- 右侧展示数据截止时间与刷新按钮

### 4.4 B区：标签及人群统计卡区

这是本次原型改造重点。

#### B区结构拆分

```text
┌────────────────────────────────────────────────────────────┐
│ 标签统计卡1 │ 标签统计卡2 │ 标签统计卡3 │ 人群统计卡          │
└────────────────────────────────────────────────────────────┘
```

#### 人群统计卡内部结构

- 只展示患者真实归属的人群
- 所有患者必有"通用人群"（主标签样式，视觉强调）
- 专项人群按实际命中情况显示 0~4 个（浅色底 + 箭头样式）

#### 人群统计卡交互要求

1. 每个人群标签都必须具备可点击感知
2. 通用人群：主标签样式，视觉强调
3. 专项人群：浅色背景 + 右侧箭头，暗示详情页尚未完整落地
4. hover 时需要有高亮反馈
5. 点击行为：
   - 通用人群：进入通用人群健康空间详情页
   - 专项人群：进入专项人群占位页

### 4.5 C区：个人健康标签图谱区

保留当前能力，本期不做改造。

## 五、页面2：通用人群健康空间详情页

### 5.1 页面定位

承载当前患者在"通用人群健康空间"下的全量健康数据。该页面是完整详情页。

### 5.2 页面整体结构

采用：**上一下二卡片网格布局**

```text
┌───────────────────────────────────────────────────────┐
│ A. 页面头部：返回入口 / 患者基础信息 / 空间标识          │
├───────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────┐   │
│ │ C1：诊疗记录（通栏）                              │   │
│ │  子菜单/搜索/列表（卡片内独立滚动）                │   │
│ └─────────────────────────────────────────────────┘   │
│ ┌─────────────────────┐  ┌─────────────────────┐     │
│ │ C2：健康史           │  │ C3：体征数据         │     │
│ │  子菜单/内容展示      │  │  子菜单/统计/趋势图   │     │
│ │  （卡片内独立滚动）    │  │  （卡片内独立滚动）    │     │
│ └─────────────────────┘  └─────────────────────┘     │
└───────────────────────────────────────────────────────┘
```

### 5.3 布局要求

- 页面头部固定在顶部
- 三模块以"上一下二"网格展示：诊疗记录通栏置顶，健康史和体征数据各占下方一半宽度
- 上下两行等高分配，各占剩余视口高度的 50%
- 每个卡片标题区固定在卡片顶部，内容区可独立滚动
- 卡片间保持清晰的视觉分隔（圆角、阴影、间距）
- 响应式适配：宽屏下上一下二网格，窄屏（≤1200px）降级为单列堆叠

### 5.4 A区：页面头部

#### 建议内容
- 返回按钮 / 面包屑
- 页面标题：通用人群健康空间
- 患者基础信息（姓名、性别、年龄、证件）
- 数据截止时间

## 六、C1：诊疗记录模块原型拆解

### 6.1 模块结构

```text
卡片标题：诊疗记录
  ├─ 子菜单：全部 / 门诊记录 / 住院记录（带数量）
  ├─ 搜索筛选区：关键词 + 时间范围（随内容滚动）
  ├─ 列表区（时间降序）
  └─ 详情弹窗（点击列表项触发）
```

### 6.2 区域说明

#### 搜索筛选区
- 关键词输入框 + 时间范围选择器 + 查询/重置按钮
- 位于列表上方，随卡片内容滚动（非固定）

#### 列表区
每条记录展示：原始诊断名称、就诊类型、就诊机构、就诊科室、就诊时间

#### 详情下钻（弹窗）
- 点击列表项 → 打开二级弹窗，展示诊疗详情/住院详情
- 二级弹窗内展示检查/检验/处方/手术记录列表
- 点击具体记录项 → 打开三级弹窗，覆盖二级弹窗
- 三级弹窗头部包含"返回"按钮（回到二级）和"关闭全部"按钮

### 6.3 原型状态建议
- 默认列表态
- 搜索筛选态
- 二级弹窗展开态（门诊详情/住院详情）
- 三级弹窗展开态（检查/检验/处方/手术详情）
- 无数据态

## 七、C2：健康史模块原型拆解

### 7.1 模块结构

```text
卡片标题：健康史
  ├─ 子菜单：既往病史 / 家族史 / 过敏史 / 个人史（带数量）
  ├─ 搜索区（仅既往病史展示）
  └─ 内容展示区
```

### 7.2 原型表达重点

#### 既往病史
- 支持关键词搜索
- 纵向列表：首次确诊时间、诊断名称、诊断机构、诊断科室

#### 家族史
- 按亲属分组的信息块

#### 过敏史
- 按过敏类型分组的信息块

#### 个人史
- 生活方式卡片组：饮食、吸烟、饮酒、运动、摄盐

### 7.3 原型状态建议
- 各子菜单展示态
- 节点缺失态（无某类数据时不展示该节点）
- 全无数据态

---

## 八、C3：体征数据模块原型拆解

### 8.1 模块结构

```text
卡片标题：体征数据
  ├─ 子菜单：血压 / 血糖 / BMI（默认选中血压）
  ├─ 统计卡区（4项统计）
  ├─ 趋势图区
  ├─ 缩放进度条占位（本期不可交互）
  └─ 溯源弹窗（点击数据点触发）
```

### 8.2 区域说明

#### 统计卡区
4 张小卡横向排布：历年检查次数、历年异常次数、近一年检查次数、近一年异常次数

#### 趋势图区
- 横坐标：日期，纵坐标：指标单位
- 点击数据点打开溯源弹窗

#### 缩放进度条
- 本期仅展示占位UI，不可交互
- 后续版本实现拖拽滑块缩放时间范围

### 8.3 原型状态建议
- 默认血压态
- 切换血糖/BMI态
- 点击数据点溯源弹窗态
- 无数据态

## 十、关键交互说明

### 10.1 本次原型必须体现的交互

1. 主界面点击"通用人群"进入详情页
2. 主界面点击"专项人群"进入占位页
3. 详情页上一下二卡片网格布局，每卡片独立滚动
4. 诊疗记录列表点击 → 二级弹窗查看详情
5. 二级弹窗内点击 → 三级弹窗查看明细
6. 体征趋势图点击查看溯源弹窗
7. 返回主界面路径

### 10.2 可延后处理的交互

1. 极端数据量下的分页/懒加载
2. 缩放进度条拖拽交互（本期仅占位）
3. 专项人群正式详情结构
