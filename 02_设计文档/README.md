# 设计文档目录

## 目录说明

存放总体架构、数据模型、指标体系、标签体系、业务规则、界面/原型设计等设计交付文档。

业务规则类文档回答“**怎么判断**”——标签命中条件、指标计算公式、字段来源与取值范围；架构与设计类文档回答“**怎么建设**”——系统分层、能力边界、数据流和设计资产。

这类文档由产品经理产出，用于研发团队统一理解系统设计、数据模型、标签规则和指标口径，是产品迭代与研发实现的设计依据。

> 与需求文档的区别：需求文档描述功能场景和交互逻辑（做什么），设计文档定义系统方案、数据结构、判断逻辑和计算规则（怎么建设、怎么判断）。
