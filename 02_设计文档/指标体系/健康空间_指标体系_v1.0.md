## 一、指标体系定位

### 1.1 指标与标签的本质区别

| 维度 | 标签 | 指标 |
|------|------|------|
| 形态 | 定性描述（高血压患者、老年人） | 定量数值（收缩压135mmHg、BMI 24.5） |
| 粒度 | 个人维度的分类结果 | 可聚合的测量值，支持个人/群体两个维度 |
| 更新频率 | 相对稳定，规则触发更新 | 随原始数据实时/定期刷新 |
| 计算逻辑 | 基于规则或模型打标 | 明确的统计计算公式 |
| 主要用途 | 人群筛选、画像分类 | 监测趋势、评估效果、支撑决策 |

> **关键关系**：指标是标签的计算基础，标签是指标的业务解读结果。
> 示例：`收缩压（指标）≥ 140mmHg 持续3次` → 打上 `高血压（标签）`

---

### 1.2 指标与原始数据的三层加工关系

```
原始数据（脏、散、多源）
    ↓  清洗 + 标准化
原子指标（最小粒度的可信测量值）
    ↓  计算 + 聚合
派生指标（基于原子指标二次计算）
    ↓  建模 + 加权
复合指标（多维度综合评估结果）
```

| 层级 | 示例 | 来源 |
|------|------|------|
| 原始数据 | HIS记录"BP 140/90"、"BP 145/88"、"BP 138/92" | HIS系统 |
| 原子指标 | 收缩压最新值：140mmHg；舒张压均值（近3次）：90mmHg | 清洗 + 提取 |
| 派生指标 | 血压控制达标率（近6个月）：62% | 原子指标计算 |
| 复合指标 | 心血管风险评分：68分 | 多个派生指标建模 |

---

### 1.3 指标在产品架构中的位置

```
原始数据层（HIS / LIS / 体检 / 用药 / 手术等）
        ↓
  【指标体系层】← 核心中间层
  计算、标准化、存储指标值
        ↓              ↓              ↓
  标签体系层       监测预警        数据共享开放
  （基于指标规则打标）  （阈值触发）     （标准化接口）
        ↓
  健康画像层（标签 + 指标综合呈现）
        ↓
  业务应用层（医生端 / 居民端 / 监管端）
```

---

### 1.4 指标的多维度应用价值

| 应用方向 | 说明 |
|---------|------|
| 支撑标签打标 | 替代各标签直接抓取原始数据，统一口径，降低维护成本 |
| 监测与预警 | 指标值 + 阈值规则，直接触发健康预警，不依赖标签 |
| 趋势分析 | 指标时序数据展示个人健康变化趋势，标签无法表达"变好/变差" |
| 干预效果评估 | 干预前后指标值对比，量化健康管理效果 |
| 数据共享开放 | 原始数据不出域，指标作为标准化共享单元，合规可用 |

## 三、指标元数据规范

每个指标的完整定义应包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 指标ID | 唯一编码 | IND-001 |
| 指标名称 | 中文业务名称 | 空腹血糖 |
| 英文代码 | 系统字段标识 | FBG |
| 所属类别 | 一级/二级分类 | 生理健康 / 实验室检验 |
| 指标层级 | 原子 / 派生 / 复合 | 原子 |
| 数据来源 | 来源系统及字段 | LIS.检验结果表 |
| 计算公式 | 提取或计算逻辑 | 直接提取最近一次检验值 |
| 时间窗口 | 取值时间范围 | 最新一次 / 近3个月 / 近1年 |
| 统计粒度 | 个人级 / 人群级 | 个人级 |
| 单位 | 数值单位 | mmol/L |
| 参考范围 | 正常值区间 | < 6.1 |
| 缺失值处理 | 无数据时的处理策略 | 标记为空，不参与计算 |
| 更新频率 | 数据刷新周期 | 就诊时触发 |
| 关联标签 | 影响哪些标签的打标规则 | 糖尿病风险标签 |
| 参考标准 | 数据口径依据 | WS/T 661-2020 |

## 四、分类特征对比

| 类别 | 数据类型 | 更新频率 | 数据质量 | 主要用途 |
|------|---------|---------|---------|---------|
| 生理健康 | 数值型 | 低频（就诊时） | 高（专业测量） | 临床判断、异常预警 |

| 综合评估 | 评分型 | 定期计算 | 依赖输入 | 风险预测、决策支持 |

## 八、与标签体系的过渡策略

### 当前状态

```
原始数据（HIS / LIS / 用药 / 手术）
    ↓ 各标签各自写规则直接抓取
标签体系（已上线，不动）
```

### 目标状态（平行建设，向前兼容）

```
原始数据
    ↓
指标层（新建）
    ↓              ↓
标签体系        新业务场景
（已有，不动）   （监测 / 共享 / 趋势）
```

### 三阶段过渡路径

| 阶段 | 动作 | 成本 | 产出 |
|------|------|------|------|
| 第一阶段：显式化 | 将标签规则中隐含的数据字段登记为指标定义，不改代码 | 极低 | 指标注册表初版 |
| 第二阶段：物化 | 对被3个以上标签共同引用的字段建中间表 | 中 | 指标可独立查询，新业务可用 |
| 第三阶段：渐进迁移 | 新增标签直接引用指标层，存量标签有改造需求时顺带迁移 | 低（分散） | 标签口径统一，维护成本下降 |
