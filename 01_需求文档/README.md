# 需求文档目录

## 目录说明

存放功能需求文档，回答"**做什么**"——功能边界、用户场景、交互流程。

这类文档是需求评审会的主要输入材料，也是产品迭代的功能依据。

> 与设计文档的区别：需求文档描述功能场景和交互逻辑，设计文档（02_设计文档）定义系统方案、判断逻辑和计算规则。
