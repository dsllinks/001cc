## 1. 产品定位与框架

### 1.1 个人数字空间定义

**个人数字空间**是健康画像系统的核心前台展示模块，以"空间"为组织单元，将居民个人的健康数据、AI生成内容（健康摘要、风险预测、个性化建议）整合呈现，帮助居民掌握自身健康状况并实现主动健康管理。

**产品边界**：
- 个人数字空间是面向**居民个人**的健康数据展示与服务平台
- 区别于「人群健康空间」（面向健康管理人员的群体管理工具）
- 区别于「健康画像（医生端）」（面向临床医生的诊前摘要工具）

**端侧策略**：
- PC端与移动端内容框架保持一致，同一居民可在两端访问
- PC端侧重信息的全局展示与详细浏览（宽屏布局）
- 移动端侧重快速查看与关键提醒（竖屏单列布局）

### 1.2 产品价值主张

| 用户群体 | 核心价值 |
|---------|---------|
| 普通居民 | 一站式了解个人健康状况，从被动就医转向主动健康管理 |
| 孕产妇 | 全周期孕产健康跟踪，覆盖孕期监测、产后恢复关键指标 |
| 婴幼儿家长 | 儿童生长发育监测，预警异常指标，辅助家长科学育儿 |
| 慢病/重疾患者 | 疾病管理专项跟踪，风险预警，个性化干预建议 |

## 2. 人群空间分类体系

### 2.1 分类体系总览

个人数字空间按**人群特征**分类，同一居民可动态归属不同人群空间：

| 空间类型 | 目标人群 | 版本 | 状态 |
|---------|---------|------|------|
| 通用数字空间 | 所有居民（默认空间） | V1.0 | 已上线 |
| 慢病人群数字空间 | 确诊慢病（高血压、糖尿病等）居民 | V1.0 | 已上线 |
| 孕产妇数字空间 | 当前怀孕或产后哺乳期女性 | V1.1 | 规划中 |
| 婴幼儿数字空间 | 0-6岁婴幼儿（由家长代为使用） | V1.1 | 规划中 |
| 重疾人群数字空间 | 确诊恶性肿瘤、心脑血管重疾等居民 | V1.1 | 规划中 |
| 自定义数字空间 | 机构按需定义的特定人群 | V1.1 | 待明确 |

### 2.2 命名规范

空间命名遵循"人群特征 + 数字空间"的格式，前台展示时可简化为"XX健康空间"：

| 通用数字空间 | 个人健康空间 |

## 3. 各人群空间内容差异矩阵

### 3.1 通用内容模块（所有空间共有）

| 模块名称 | 内容说明 | 数据来源 |
|---------|---------|---------|
| 基本信息 | 姓名、性别、年龄、签约医生 | 患者基础档案 |
| 健康标签云 | 慢病标签、风险等级标签、生活习惯标签 | 标签知识库 + 模型标签 |
| AI健康摘要 | 通俗化的个人健康状况总结（大模型生成） | 讯飞医疗大模型 |
| 核心健康指标 | 血压、血糖、BMI、体温等关键指标最新值 | HIS系统 + 健康档案 |
| 个性化建议 | 饮食建议、运动建议（大模型生成，流式输出） | 讯飞医疗大模型 |
| 健康风险评估 | 主要疾病风险等级（低/中/高） | AI风险评估模型 |

### 3.2 各人群空间专项模块

## 4. 人群归属逻辑

### 4.1 归属判断规则

居民的人群归属由系统根据**健康标签**自动判断，无需居民手动选择：

| 判断维度 | 判断依据 | 归属空间 |
|---------|---------|---------|
| 妊娠状态 | 含"当前妊娠"或"产后哺乳期"标签 | 孕产妇数字空间 |
| 年龄段 | 年龄 0-6岁（儿童档案） | 婴幼儿数字空间 |
| 重大疾病诊断 | 含恶性肿瘤、心梗、脑卒中、慢性肾病等重疾诊断标签 | 重疾人群数字空间 |
| 慢病诊断 | 含高血压、糖尿病、冠心病、慢阻肺等慢病标签 | 慢病人群数字空间 |
| 无以上特征 | 默认 | 通用数字空间 |

### 4.2 多人群重叠处理策略

**推荐方案**：优先级归属 + 多空间并行访问

居民可同时存在于多个人群空间，但系统以**优先级**决定默认展示的主空间：

```
优先级顺序（由高到低）：
1. 孕产妇数字空间（妊娠/产后状态优先）
2. 婴幼儿数字空间（儿童年龄段优先）
3. 重疾人群数字空间（重大疾病诊断优先）
4. 慢病人群数字空间（慢病诊断次之）
5. 通用数字空间（兜底默认）
```

**典型重叠场景处理示例**：

| 场景 | 标签组合 | 归属结果 |
|-----|---------|---------|
| 孕期糖尿病患者 | 当前妊娠 + 糖尿病 | 主空间：孕产妇数字空间（孕期优先）；慢病空间可切换访问 |
| 合并高血压的肿瘤患者 | 恶性肿瘤 + 高血压 | 主空间：重疾人群数字空间；慢病空间可切换访问 |
| 普通高血压患者 | 高血压 | 主空间：慢病人群数字空间 |

**前台切换机制**：当居民归属多个空间时，空间顶部展示"切换空间"入口，居民可手动切换查看其他人群空间视角。

### 4.3 人群身份动态演变机制

人群归属状态随居民健康数据实时更新：

| 状态变化 | 触发条件 | 空间变化 |
|---------|---------|---------|
| 进入孕产妇空间 | 产检档案新增"当前妊娠"标签 | 自动切换为孕产妇空间为主空间 |
| 离开孕产妇空间 | 产后满12个月或哺乳期结束标签更新 | 回归慢病/通用空间 |
| 进入重疾空间 | 新增重疾诊断标签 | 自动切换为重疾空间为主空间 |
| 婴幼儿转通用 | 儿童年满7岁 | 自动切换为通用/慢病空间 |

## 5. PC端界面适配策略

### 5.1 PC端核心优势场景

PC端相比移动端在以下场景具有体验优势，需重点强化：
- **图表阅读**：趋势图、生长曲线图等需要大屏展示的内容
- **详细信息浏览**：历史记录、详细检查报告等长文本内容
- **多模块对比**：同时查看多个健康指标维度

### 5.2 PC端通用布局规范

**整体框架**：

```
┌───────────────────────────────────────────────────┐
│  顶部导航栏：系统Logo + 居民姓名 + 空间切换 + 登出     │
├───────┬───────────────────────────────────────────┤
│       │  主内容区（右侧占主屏幕 80%）                 │
│ 左侧   ├───────────────────────────────────────────┤
│ 导航   │  内容区 Section 1：个人信息 + AI摘要           │
│ 菜单   ├───────────────────────────────────────────┤
│（20%） │  内容区 Section 2：核心指标卡片组              │
│       ├───────────────────────────────────────────┤
│       │  内容区 Section 3：专项模块（人群差异化内容）   │
│       ├───────────────────────────────────────────┤
│       │  内容区 Section 4：建议与计划                 │
└───────┴───────────────────────────────────────────┘
```

**左侧导航菜单内容**（通用）：
- 我的健康概览
- 健康指标
- 检查/检验记录
- 随访记录
- 健康建议
- （人群专属）孕期管理 / 儿童档案 / 疾病管理

### 5.3 PC端与移动端差异说明

| 功能模块 | PC端呈现方式 | 移动端呈现方式 |
|---------|------------|-------------|
| AI健康摘要 | 左侧固定展示，全文展开 | 折叠展示，点击展开 |
| 健康指标卡片 | 6-8个指标并列展示（3列×2行） | 2个指标并列（2列×N行，可滑动） |
| 趋势图表 | 完整趋势图，支持时间范围筛选 | 简化折线图，默认近30天 |
| 检查记录列表 | 表格形式，多列展示 | 卡片列表，竖向滚动 |
| 建议内容 | 三栏并列展示（饮食/运动/其他） | 单列卡片，逐项展示 |

## 6. 数据流转示意

```
数据输入层
  HIS系统（就诊/检查/检验/处方）
  区域卫生平台（体检/公卫/随访）
  儿保系统（生长发育/疫苗）
  产检系统（产检记录/B超报告）
        │
        ▼
数据加工层（ODS → DWD）
  数据标准化、清洗、指标计算
        │
        ▼
标签生成层（ADS）
  规则标签：慢病诊断、年龄性别、妊娠状态
  模型标签：风险等级预测、人群归属判断
  AI生成内容：健康摘要、个性化建议（讯飞医疗大模型）
        │
        ▼
生成调度层
  定时刷新：健康摘要、风险标签（每日/每次就诊后刷新）
  实时生成：医生查看诊前摘要时触发
        │
        ▼
个人数字空间展示层
  根据人群标签匹配对应空间
  按优先级确定主空间
  前台渲染展示（PC端 / 移动端）
        │
        ▼
共享开放层
  页面集成（居民ID + 空间类别）→ 嵌入第三方平台
  数据接口（居民ID + 空间类别）→ 数据互通
```
