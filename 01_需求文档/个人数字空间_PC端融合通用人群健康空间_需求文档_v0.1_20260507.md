## 1. 背景与目标

当前 PC 端个人数字空间已具备个人信息、标签统计和个人健康标签图谱能力，可从个体视角查看患者命中的健康标签、风险标签和标签类别。

下一版本需要在 PC 端将个人数字空间与人群健康空间进行融合，使健康管理人员在查看单个患者时，不仅能看到该患者的标签数据，也能查看其在通用人群空间下沉淀的健康特征数据，例如诊疗记录、健康史、体征指标趋势和年度账单。

本需求仅考虑“通用人群健康空间”融合。由于同一患者必定属于通用人群，因此 PC 端默认展示通用人群空间内容；孕产妇、慢病、幼儿、重疾等专项人群仅作为后续扩展方向，本期不展开详情设计。

### 1.1 建设目标

- 保留当前 PC 端个人数字空间的个人健康标签图谱能力。
- 在个体视角下融合通用人群健康空间数据。
- 尽可能复用移动端通用人群健康空间既有模块和业务规则。
- 支持健康管理人员在一个页面内完成“看标签、看事件、看趋势、看消费”的综合判断。

### 1.2 非目标

- 本期不设计孕产妇、慢病、幼儿、重疾专项人群空间详情。
- 本期不重做人群筛选、患者列表、同群对比和人群管理工作台。
- 本期不改变移动端通用人群健康空间的数据口径和业务规则。
- 本期不制作独立原型图或修改现有 HTML 原型。

## 2. 当前界面分析

当前 PC 端个人数字空间页面主要包含以下模块：

| 区域 | 当前能力 | 本期处理 |
|------|---------|---------|
| 顶部个人信息区 | 展示姓名、性别、年龄、民族、证件、联系方式、地址、数据截止时间 | 保留 |
| 标签统计卡片 | 展示标签类别、命中标签数、高风险标签数 | 保留 |
| 个人健康标签图谱 | 以图谱方式展示患者命中的标签及标签关系 | 保留为主视觉 |
| 标签图例 | 展示标签类别、可展开标签、不可展开标签、可下钻标签说明 | 保留 |

本期新增“通用人群健康空间”区域，用于承载移动端通用人群健康空间的四个标签页能力。

## 3. 本期范围

### 3.1 融合对象

本期融合对象为当前患者的“通用人群健康空间”。

患者归属规则：

- 同一患者必定属于通用人群。
- 患者可能同时属于孕产妇、慢病、幼儿或重疾等专项人群。
- 本期仅展示通用人群空间详情。
- 专项人群可在页面中展示归属状态或预留入口，但不展示专项空间内容。

### 3.2 复用模块

PC 端复用移动端通用人群健康空间的四个模块：

| 模块 | 移动端能力 | PC端融合要求 |
|------|-----------|-------------|
| 诊疗记录 | 全部、门诊记录、住院记录；搜索筛选；详情下钻 | 复用业务规则，适配宽屏列表和详情展示 |
| 健康史 | 既往病史、家族史、过敏史、个人史 | 复用分类、搜索和空数据规则 |
| 体征数据 | 血压、血糖、BMI统计与趋势图 | 复用统计、趋势、缩放和溯源规则 |
| 年度账单 | 年份切换；门诊、住院、体检消费统计 | 复用年份切换和分类统计 |

## 4. 页面结构

PC 端融合页面建议分为四个区域：

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部个人信息区：个人基础信息 / 数据截止时间 / 刷新             │
├────────────────────────────────────────────────────────────┤
│ 标签统计区：标签类别 / 命中标签数 / 高风险标签数 / 通用人群归属 │
├────────────────────────────────────────────────────────────┤
│ 个人健康标签图谱区：标签图谱 / 标签类别筛选 / 图例说明          │
├────────────────────────────────────────────────────────────┤
│ 通用人群健康空间区：诊疗记录 / 健康史 / 体征数据 / 年度账单 Tab │
└────────────────────────────────────────────────────────────┘
```

### 4.1 顶部个人信息区

保留当前展示内容：

- 姓名
- 性别
- 年龄
- 民族
- 证件信息
- 联系方式
- 地址
- 数据截止时间
- 刷新入口

### 4.2 标签统计区

保留当前统计卡片：

- 标签类别
- 命中标签数
- 高风险标签

新增通用人群归属展示：

- 展示“通用人群”归属标识。
- 若患者属于孕产妇、慢病、幼儿、重疾等专项人群，可展示专项人群标签，但本期不支持点击进入详情。

### 4.3 个人健康标签图谱区

保留当前图谱能力：

- 支持按标签类别筛选图谱。
- 支持查看可展开、不可展开、可下钻标签。
- 支持展示管理标签集可见范围。
- 图谱节点、颜色、连线和下钻逻辑不在本期重做。

### 4.4 通用人群健康空间区

在标签图谱下方或右侧新增通用人群健康空间区域。

该区域使用 Tab 组织：

- `诊疗记录`
- `健康史`
- `体征数据`
- `年度账单`

PC 端默认进入 `诊疗记录` Tab。

## 5. 功能需求

### 5.1 诊疗记录

#### 功能说明

展示患者全量诊疗数据，支持按记录类型、关键词和时间范围查看。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 全部 | 展示全部门诊和住院记录 |
| 门诊记录 | 仅展示门诊相关记录 |
| 住院记录 | 仅展示住院相关记录 |

每个菜单名称后展示该菜单下的数据量。

#### 搜索筛选

- 支持关键词模糊搜索。
- 支持时间范围筛选。
- 关键词搜索和时间范围筛选可组合使用。

#### 列表展示

列表项按时间降序排列。

每条列表展示：

- 原始诊断名称
- 就诊类型：门诊 / 住院
- 就诊机构
- 就诊科室
- 就诊时间

#### 详情下钻

- 点击门诊记录，下钻展示 `诊疗详情`。
- 点击住院记录，下钻展示 `住院详情`。
- PC 端可采用弹窗、抽屉或详情区展示，详情字段与移动端保持一致。

### 5.2 健康史

#### 功能说明

展示患者既往健康数据。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 既往病史 | 展示患者既往诊断记录 |
| 家族史 | 展示亲属关系及家族病史 |
| 过敏史 | 展示过敏类型和过敏源 |
| 个人史 | 展示饮食、吸烟、饮酒、运动、摄盐等生活方式信息 |

每个菜单名称后展示该菜单下的数据量。

#### 搜索规则

- `既往病史`、`家族史`、`过敏史`支持关键词模糊搜索。
- `个人史`本期不要求搜索。

#### 展示规则

- 既往病史按时间降序排列。
- 既往病史列表项展示首次确诊时间、诊断名称、诊断机构、诊断科室。
- 家族史展示亲属关系和原始诊断名称。
- 过敏史展示过敏类型和过敏源名称。
- 个人史展示饮食习惯、吸烟情况、饮酒情况、运动情况、摄盐情况。

#### 空数据规则

- 无某类亲属家族史时，不展示该亲属节点。
- 无某类过敏史时，不展示该过敏类型节点。
- 无某类个人史数据时，不展示该个人史节点。
- 当前菜单无任何数据时，显示 `暂无数据`。

### 5.3 体征数据

#### 功能说明

展示患者体征指标的统计信息和趋势变化。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 血压 | 展示血压统计和趋势 |
| 血糖 | 展示血糖统计和趋势 |
| BMI | 展示 BMI 统计和趋势 |

#### 统计信息

每个体征指标下展示：

- 历年来检查次数
- 历年来异常次数
- 近一年来检查次数
- 近一年来异常次数

#### 趋势图

- 横坐标为日期。
- 纵坐标为指标单位。
- 默认展示三个日期。
- 趋势图下方展示缩放进度条。
- 支持通过缩放进度条调整横坐标展示范围。

#### 溯源弹窗

- 点击趋势图数据点后，弹窗展示该数据点的溯源信息。
- 溯源字段与 PC 端既有溯源字段保持一致。
- 弹窗支持关闭后回到当前体征指标趋势图。

### 5.4 年度账单

#### 功能说明

统计居民年度医疗消费数据。

#### 年份切换

- 支持切换年份查看。
- 默认选中当年年份。

#### 消费分类

年度账单分为三类：

| 分类 | 说明 |
|------|------|
| 门诊 | 包含门诊和急诊消费 |
| 住院 | 统计住院消费 |
| 体检 | 统计体检消费 |

## 6. PC端适配规则

### 6.1 布局适配

- 移动端纵向列表在 PC 端可改为宽屏列表。
- 搜索筛选区固定在列表上方。
- 列表与详情可采用弹窗、抽屉或左右分区展示。
- 图谱区与通用人群健康空间区需保持层次清晰，避免在同一屏内信息过载。

### 6.2 业务规则复用

- 诊疗记录、健康史、体征数据、年度账单的数据口径沿用移动端。
- PC 端仅做布局适配，不重新定义统计口径。
- 原始诊断名称、原始机构、原始科室等字段沿用移动端展示口径。

### 6.3 默认展示

- 页面默认展示当前患者的个人健康标签图谱。
- 通用人群健康空间区域默认选中 `诊疗记录`。
- 体征数据默认选中 `血压`。
- 年度账单默认选中当年年份。

### 6.4 专项人群预留

- 若患者属于孕产妇、慢病、幼儿、重疾等专项人群，可在归属展示区展示标签。
- 专项人群标签本期不进入详情页。
- 可展示“后续建设”或置灰状态，避免用户误以为已支持专项空间。
