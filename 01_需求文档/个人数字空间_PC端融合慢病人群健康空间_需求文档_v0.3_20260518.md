### v0.2 相对 v0.1 的核心差异

| 编号 | 维度 | v0.1 | v0.2 |
|------|------|------|------|
| D1 | 病种卡滚动行为 | 第3层病种卡随页面滚动一并滚动 | 滚动后收缩为标签条（"高血压 \| 糖尿病"）悬浮在卡片区顶部，保持病种上下文始终可见 |
| D2 | 药物管理展示形式 | 宽屏表格展示 | 改为列表形式（每条展示药品名+剂量+频次等关键字段），点击展开完整字段弹窗，与其他子菜单交互形态统一 |
| D3 | 模块一"全部"数据边界 | 描述模糊："所有慢病相关诊疗记录、用药记录、并发症记录" | 明确为"仅包含诊疗记录（HIS来源）"，不包含公卫随访记录；随访记录统一在模块三展示 |
| D4 | 并发症识别时间窗 | "首诊时间之后的所有并发症" | 明确为"首诊时间当天及之后"（含首诊当天），避免首诊日发现的并发症被漏识 |

| D6 | 单病种患者指标子菜单 | 始终展示血压/血糖/血脂三项 | 按病种筛选：仅高血压→血压+血脂，仅糖尿病→血糖+血脂，双病种→三项全展示 |
| D7 | 随访记录定位能力 | 仅按时间降序展示 | 增加年份快速跳转（表格顶部年份下拉框），长期患者可快速定位特定年份的随访 |

### 保留项（未做变更）

- 专病档案卡片信息密度：维持 v0.1 基础摘要形式，暂不前置控制目标或合并随访
- 前期风险异常值高亮：维持统一橙色，不做血压 130-134 / 135-139 分层
- 病种卡视觉权重：两张病种卡并排展示，不区分主病种与合并病种

### 变更原因

v0.1 评审中发现病种上下文容易在滚动中丢失、药物管理交互与其他子菜单形态差异过大、"全部"数据边界模糊可能与随访记录重叠、单病种患者存在长期无数据的"血糖/血压"空子菜单等问题。v0.2 针对这 7 项做定向优化，保持整体章节结构和布局方案不变。

### 核心设计原则（沿用 v0.1）

- 章节结构严格对齐通用人群 PC 端 v0.3（10 章结构）。
- 详情页布局沿用通用人群 v0.3 的 2×2 卡片网格，4 模块一页平铺。
- 业务规则完全复用移动端慢病人群健康空间，PC 端仅做布局适配。
- 主界面不做任何改动，仅升级慢病标签的视觉样式和跳转目标。

## 1. 背景与目标

### 1.1 背景

通用人群健康空间 PC 端融合需求（v0.3）已完成定稿，主界面三区布局及人群卡片入口机制已确定。当前慢病人群在主界面人群卡片中作为专项人群标签存在，采用"浅色底+箭头"弱视觉样式，点击后跳转至占位页（提示"详情页后续开放"）。

本期需求承接 v0.3 的预留扩展机制，正式落地"慢病健康空间"详情页：

- 移动端慢病人群健康空间已上线，业务规则成熟，覆盖高血压、糖尿病两类慢病。
- V1.1 版本规划将"慢病人群"作为重点人群之一，需要在 PC 端为健康管理人员提供完整的慢病视图。
- 主界面布局保持不变，仅本期将慢病标签的跳转目标从"占位页"切换为"慢病健康空间详情页"，并将标签视觉样式从弱视觉升级为正常可点击样式。

### 1.2 建设目标

- 复用主界面三区布局，**主界面不做任何改动**（仅升级慢病标签视觉样式和跳转目标）。
- 落地"慢病健康空间详情页"，承载当前患者的慢病全量健康数据。
- 沿用通用人群详情页的 2×2 卡片网格布局，一页平铺四模块。
- 复用移动端慢病人群健康空间的业务规则（前期风险阈值、初诊确诊判定、病种分型、并发症识别、用药记录、慢病随访等）。
- 围绕健康管理人员视角组织模块顺序，强调诊疗阶段演进、指标趋势和档案管理。
- 支持高血压、糖尿病两类慢病的共存展示。

### 1.3 非目标

- 本期不涉及主界面布局调整。
- 本期不重做移动端慢病人群健康空间的业务规则和数据口径。
- 本期不覆盖除高血压、糖尿病之外的其他慢病病种（如冠心病、脑卒中、COPD 等），后续版本再扩展。
- 本期不实现"未建档引导居民建档"的居民端交互（PC 端为健康管理人员视角，仅展示建档状态）。

## 2. 范围与边界

### 2.1 融合对象

本期融合对象为当前患者的"慢病健康空间"。

患者归属规则：
- 仅满足展示条件（见第 6 章）的患者，慢病标签才会出现在主界面人群卡片中。
- 慢病标签一旦命中即**长期保留**，不设关闭条件（详见 6.4 节）。
- 详情页内容按病种组织，支持**高血压、糖尿病单独命中或同时命中**三种情况；两类慢病共存时，数据范围合并展示，页面头部并排展示两张病种卡。

### 2.2 复用模块

PC 端复用移动端慢病人群健康空间的核心业务规则：

| 移动端模块              | 移动端能力                                           | PC 端融合要求                                |
| ------------------ | ----------------------------------------------- | --------------------------------------- |
| 慢病健康视图 - 前期风险（2.1） | 首次确诊前血压 130-139/85-89、血糖 6.1-7.0 的诊疗/体检记录；异常值高亮 | 复用业务规则，合并至 PC 端"慢病健康视图"模块作为子菜单          |
| 慢病健康视图 - 初诊确诊（2.2） | 最早一条高血压/糖尿病诊断记录；基于疾病知识库 30+ 分型识别                | 复用业务规则，合并至 PC 端"慢病健康视图"模块作为子菜单          |
| 慢病健康视图 - 药物管理（2.3） | 公卫表高血压/糖尿病用药记录，按时间倒序                            | 复用业务规则，合并至 PC 端"慢病健康视图"模块作为子菜单，改为宽屏表格展示 |
| 慢病健康视图 - 复发控制（2.4） | 首诊之后所有并发症诊断记录                                   | 复用业务规则，合并至 PC 端"慢病健康视图"模块作为子菜单，支持弹窗详情下钻 |
| 健康指标               | 血压、血糖、血脂趋势图                                     | 复用业务规则，对应 PC 端"健康指标"模块                  |
| 随访记录               | 慢病随访表列表，点击查看详情                                  | 复用业务规则，对应 PC 端"随访记录"模块                  |
| 健康档案               | ADS 专病档案                                        | 复用业务规则，对应 PC 端"专病档案"模块                  |

**PC 端模块重组说明**：移动端 7 大模块在 PC 端合并重组为 4 个卡片模块（详见 5.3 节布局），以适配 2×2 网格的容量约束。

### 2.3 不改动项

- 个人数字空间主界面布局和交互（沿用 v0.3 三区布局）。
- 主界面人群卡片的展示规则、可点击标识规则、跳转行为表。
- 通用人群、孕产妇等其他人群的展示规则与跳转行为。
- 移动端慢病人群健康空间的业务规则和数据口径。

## 4. 主界面变更说明

### 4.1 整体说明

个人数字空间主界面**完全沿用 v0.3 布局**，本期仅对"慢病"标签做两项调整：

1. **视觉样式升级**：从 v0.3 的"浅色底+箭头"弱视觉样式，升级为与通用人群标签等同的正常可点击样式。
2. **跳转目标切换**：从"占位页（提示详情页后续开放）"切换为"慢病健康空间详情页"。

主界面其他区域（个人信息区、标签统计部分、标签图谱区）完全不变。

### 4.3 主界面展示条件提示

- 慢病标签是否出现在主界面人群卡片中，由第 6 章展示条件判定。
- 患者未命中高血压/糖尿病标签时，慢病标签不出现，对应的跳转入口也不展示。
- 患者同时命中高血压和糖尿病时，主界面仍只展示一个"慢病"标签（不拆分为两个），详情页内再展开两个病种。

## 5. 慢病健康空间详情页

### 5.1 页面入口与定位

**入口**：个人数字空间主界面 → 标签及人群统计卡 → 点击"慢病"标签。

**页面定位**：独立详情页，承载当前患者的慢病全量健康数据，PC 端一页平铺展示。

**页面范围**：
- 覆盖当前患者命中的慢病病种（高血压、糖尿病，或两者兼有）。
- 覆盖从前期风险识别到随访管理、档案管理的慢病全流程数据。

### 5.2 页面头部（含慢病病种卡）

页面头部分为三层，类比通用人群 v0.3 的两层结构，新增第 3 层"慢病病种卡"以承载慢病核心上下文。

#### 第 1 层：导航条

- **返回入口**：面包屑/返回按钮，支持返回个人数字空间主界面。
- **当前空间标识**：明确显示"慢病人群健康空间"。

#### 第 2 层：患者基础信息

与通用人群详情页保持一致：
- 姓名、性别、年龄、证件信息
- 数据截止时间

#### 第 3 层：慢病病种卡（慢病专属）

**展示规则**：
- 患者仅患有一种慢病时，展示一张病种卡。
- 患者同时患有高血压和糖尿病时，并排展示两张病种卡。

**病种卡字段**：

| 字段   | 说明                                 | 数据来源         |
| ---- | ---------------------------------- | ------------ |
| 病种名称 | 高血压 / 糖尿病                          | 居民标签         |
| 建档时间 | 专病档案建档日期                           | ADS 专病档案     |
| 家庭医生 | 签约家庭医生姓名                           | 公卫签约数据       |
| 病种分型 | 基于疾病知识库识别的具体分型（如"原发性高血压"、"2 型糖尿病"） | 疾病知识库 + 最新诊断 |

**展示形式**：
```text
┌──────────────────────────────────────────────────────────────┐
│ 第 1 层：← 返回  |  慢病健康空间                               │
├──────────────────────────────────────────────────────────────┤
│ 第 2 层：姓名 性别 年龄 证件号  |  数据截止时间：YYYY-MM-DD     │
├──────────────────────────────────────────────────────────────┤
│ 第 3 层：慢病病种卡                                            │
│   ┌────────────────────┐  ┌────────────────────┐            │
│   │ ❤ 高血压            │  │ 🩸 糖尿病           │            │
│   │ 建档：2020-03-15    │  │ 建档：2021-06-20   │            │
│   │ 家医：张医生        │  │ 家医：李医生       │            │
│   │ 分型：原发性高血压   │  │ 分型：2 型糖尿病    │            │
│   └────────────────────┘  └────────────────────┘            │
└──────────────────────────────────────────────────────────────┘
```

#### 头部固定行为

- 第 1 层导航条固定不滚动。
- 第 2 层患者基础信息随页面滚动一并滚动，避免遮挡。
- 第 3 层慢病病种卡：**滚动后收缩为标签条悬浮在卡片区顶部**，保持病种上下文始终可见。
  - 默认完整展示态：展示完整病种卡（含建档时间、家医、分型等字段），与第2层共同滚动。
  - 滚动收缩态：当卡片区开始滚动后，第3层收缩为一行标签条（如 `[高血压] [糖尿病]`），悬浮固定在卡片区顶部，仅展示病种名称。
  - 反向滚动恢复：滚动到顶部时，标签条自动展开为完整病种卡。
  - 标签条点击行为：点击标签条任一病种标签，平滑展开为完整病种卡。
- 卡片区域占据剩余视口高度，四模块 2×2 网格在其中独立滚动。

### 5.3 整体布局

**核心原则**：沿用通用人群 v0.3 的卡片式 2×2 网格布局，四模块一页平铺，无 Tab 切换。

**布局方案**：卡片式网格布局（2×2）

```text
┌───────────────────────────────────────────────────────┐
│ 页面头部（导航条 / 患者信息 / 慢病病种卡）               │
```

### 5.4 模块一：慢病健康视图

#### 功能说明

以诊疗阶段为主线，展示患者的慢病发展全过程——从前期风险识别、初诊确诊、到药物管理、复发控制的完整演进。本模块合并移动端 2.1~2.4 节能力，通过子菜单切换不同阶段视图。

#### 子菜单（按疾病阶段）

| 菜单   | 对应移动端节 | 数据范围                                                   |
| ---- | ------ | ------------------------------------------------------ |
| 前期风险 | 2.1    | 首次确诊前，血压 130-139/85-89 mmHg 或空腹血糖 6.1-7.0 mmol/L 的**就诊记录**（仅 HIS 来源诊疗记录，不包含体检数据、随访记录） |
| 初诊确诊 | 2.2    | 最早一条诊断为高血压/糖尿病的就诊记录，以及疾病知识库识别的具体分型                     |
| 药物管理 | 2.3    | 公卫表中的高血压/糖尿病用药记录                                       |
| 复发控制 | 2.4    | **首诊时间当天及之后**（含首诊当天），所有高血压/糖尿病并发症诊断的就诊记录               |

每个菜单名称后展示该菜单下的数据量。

默认选中 `前期风险` 菜单。点击不同菜单时，列表区域只显示对应分类的记录，实现真实过滤。

#### 搜索筛选

- 支持关键词模糊搜索（搜索范围：诊断名称、药品名称、机构名称）。
- 支持时间范围筛选。
- 关键词搜索和时间范围筛选可组合使用。
- 搜索筛选区位于列表上方，随内容滚动。
- **前期风险、初诊确诊、药物管理、复发控制** 四个子菜单下，搜索筛选范围限定在当前子菜单数据内。

#### 列表展示

按不同子菜单展示不同列表结构：

**前期风险 / 初诊确诊 / 复发控制 子菜单下（诊疗记录列表）**：

列表项按时间降序排列，每条展示：
- 原始诊断名称
- 就诊类型：门诊 / 住院
- 就诊机构
- 就诊科室
- 就诊时间
- **异常值标识**（仅前期风险子菜单下）：对接近异常的血压值 130-139/85-89、血糖值 6.1-7.0 做高亮展示（橙色背景+加粗）

> 前期风险范围仅含 HIS 来源的就诊记录（门诊/住院），不纳入体检数据、公卫随访、自测数据等其他来源，避免与模块二「健康指标」「模块三随访记录」数据重叠。

**药物管理 子菜单下（用药记录列表）**：

采用列表形式展示，与其他子菜单交互形态保持统一。每条列表项展示关键字段：

- 药品名称（加粗主字段）
- 剂量 + 频次（如"5mg / 每日 1 次"）
- 用药起始时间
- 用药来源标识（公卫表/处方）

按用药时间降序排列。

#### 详情下钻

- **前期风险 / 初诊确诊 / 复发控制 子菜单**：点击列表中的诊疗记录，以弹窗（Modal）形式展示诊疗详情（门诊/住院），交互方式与通用人群 v0.3 保持一致。

**诊疗详情弹窗字段补充**：

在就诊详情区块中，"诊断名称"字段之后新增"一般检查"字段，展示血压和空腹血糖数据。异常值用红色高亮显示（色值 #dc2626）。

示例值：
- 血压 139/88 mmHg（红色）
- 空腹血糖 6.1–7.0 mmol/L（红色）

#### 三级下钻（检查/检验/处方/手术）

**完全复用通用人群 v0.3 的三级下钻规则，包括展示内容及交互方式**

#### 空数据规则

- 当前子菜单无任何数据时，列表区展示 `暂无数据`。
- 患者未到达某阶段（如尚未出现并发症）时，"复发控制"子菜单展示 `暂无并发症记录`。
- 患者直接确诊未经前期风险阶段时，"前期风险"子菜单展示 `未命中前期风险阈值`。

### 5.5 模块二：健康指标

#### 功能说明

展示患者血压、血糖、血脂三大慢病核心指标的统计信息和趋势变化。本模块对应移动端"3 健康指标"章节，在通用人群 v0.3"体征数据"模块基础上做慢病化调整。

#### 子菜单

子菜单根据患者实际命中的慢病病种动态展示：

| 患者命中病种 | 展示子菜单 | 默认选中 |
|------------|----------|---------|
| 仅高血压 | 血压 / 血脂 | 血压 |
| 仅糖尿病 | 血糖 / 血脂 | 血糖 |
| 高血压 + 糖尿病 | 血压 / 血糖 / 血脂 | 血压 |

各子菜单说明：

| 菜单  | 说明                                 |
| --- | ---------------------------------- |
| 血压  | 展示血压统计和趋势                          |
| 血糖  | 展示血糖统计和趋势                          |
| 血脂  | 展示血脂统计和趋势（包含总胆固醇、甘油三酯、LDL-C、HDL-C） |

单病种患者的非关联指标长期无数据意义有限（如仅高血压患者的血糖指标），按病种筛选子菜单避免空数据子菜单干扰管理视角。血脂作为两类慢病的共同关注指标，始终展示。

#### 统计信息

每个指标下展示四项统计：
- 历年来检查次数
- 历年来异常次数
- 近一年来检查次数
- 近一年来异常次数

> 不再单独展示「确诊后检查 / 确诊后异常」两项统计。原因：「确诊后」起算点依赖各病种首诊时间，对未确诊或多病种患者口径复杂，且与历年统计形成重复信息；本期由"历年 + 近一年"双维度承载长期与近期视角即可。

#### 趋势图

- 横坐标为日期。
- 纵坐标为指标单位。
- 趋势图下方展示缩放进度条占位。
- PC 端趋势图宽度适配宽屏，可展示更多日期数据点。

> 本期不在趋势图中叠加控制目标参考线、不对数据点做"是否超过控制目标"的高亮判断。原因：慢病人群血压、血糖、血脂的个体化控制目标尚未由医疗专家确认（参见第 11 章 Q3），任何阈值线均可能被误读为业务规则；待控制目标议题闭环后再补充。

#### 溯源弹窗

- 点击趋势图数据点后，弹窗展示该数据点的溯源信息。
- 溯源字段与健康画像 PC 端既有溯源字段保持一致。
- 弹窗支持关闭后回到当前指标趋势图。

- 宽屏表格，字段对应上述随访记录列表项。

- **表格顶部增加年份快速跳转下拉框**（默认展示当前年份），选中后跳转到该年份首条随访记录，便于长期患者快速定位特定年份记录。

**年份跳转交互规则**：

- 选中年份后，表格滚动到该年份的首条记录（即该年份最近一次随访）。
- 选中年份不影响数据范围（仍展示全部随访记录），仅做定位。

#### 详情下钻

点击随访记录，以弹窗形式展示完整随访详情。弹窗宽度 880px，内容分为五个区块：

**1. 随访详情**

- 下次随访日期
- 随访状态标签（如"规律随访"）

**2. 核心指标**
- 收缩压/舒张压（空腹血糖/餐后2小时血糖）：蓝色渐变大卡片（背景 `linear-gradient(135deg,#3b82f6,#2563eb)`），白色文字，显示数值和单位，根据随访类型展示血压或血糖指标。
- 体重、BMI、心率：3 列网格小卡片，居中显示数值

**3. 生活方式目标**

- 每行显示"当前值"和"目标值"对比（3 列网格：标签 90px / 当前值 / 目标值）
- 浅灰背景行项

**4. 用药**
- 服药依从性、不良反应（2 列网格）
- 药品卡片列表：药品名称、用法用量、服用时间

**5. 医生健康指导**
- 一段指导文字，浅灰背景文本框

#### 空数据规则

- 无随访记录时，展示 `暂无随访记录`。

## 6. 展示条件

### 6.1 展示条件（慢病标签出现的前提）

**主界面慢病标签出现的前提**：人群分类标签命中高血压人群或糖尿病人群中的**任意一个**。

### 6.2 病种判断规则

**高血压病种识别**：
基于疾病知识库，以下诊断名称均识别为高血压病种（移动端原文保留）：
- 原发性高血压、高血压病、高血压 1 级（低危/中危/高危/很高危）、高血压 2 级（低危/中危/高危/很高危）、高血压 3 级（低危/中危/高危）
- 单纯收缩期高血压、白大衣高血压、临界性高血压、高原性高血压、假性高血压、应激性高血压、低肾素性高血压
- 继发性高血压（含肾性高血压、肾血管性高血压、肾实质性高血压、嗜铬细胞瘤性高血压、原发性醛固酮增多症性高血压、大动脉炎性高血压、继发于内分泌疾患的高血压、阻塞性睡眠呼吸暂停低通气综合征性高血压等）
- 恶性高血压、高血压脑病、高血压危象、高血压急症、高血压亚急症
- 老年收缩期高血压、青春期高血压、难治性高血压、H 型高血压
- 糖耐量受损伴高血压、肾萎缩伴高血压、糖耐量受损伴肥胖型高血压

**糖尿病病种识别**：
基于疾病知识库，以下诊断名称均识别为糖尿病病种（移动端原文保留）：
- 糖尿病、1 型糖尿病、1 型糖尿病伴血糖控制不佳
- 2 型糖尿病、2 型糖尿病不伴有并发症、2 型糖尿病伴血糖控制不佳、2 型糖尿病性胰岛素抵抗
- 特殊类型糖尿病、类固醇性糖尿病、胰源性糖尿病、肝移植术后糖尿病、脆性糖尿病、线粒体糖尿病、潜伏性糖尿病、继发性糖尿病、肝性糖尿病
- 糖尿病伴有昏迷、糖尿病性低血糖症、糖尿病性低血糖昏迷、糖尿病性前期微白蛋白尿、糖尿病性胰岛素抵抗、糖尿病性潮红
- 青年起病成年型糖尿病、青少年 2 型糖尿病、成人隐匿性自身免疫性糖尿病
- 糖尿病不伴有并发症、糖尿病不伴有并发症（其他特指的）、糖尿病伴血糖控制不佳

### 6.3 高血压与糖尿病的并存展示

- 患者仅患高血压或仅患糖尿病时，详情页头部展示 1 张病种卡，模块一、二、三、四数据范围限定在该病种。
- 患者同时患有高血压和糖尿病时，详情页头部并排展示 2 张病种卡，模块一、二、三、四数据范围合并为两类慢病的并集。
- 模块一的"初诊确诊"子菜单展示两类病种的初诊记录（两条记录分别对应高血压和糖尿病首诊）。
- 模块一的"药物管理"子菜单合并展示两类慢病用药。
- 模块一的"复发控制"子菜单合并展示两类慢病的并发症。
- 模块二的"健康指标"按指标类型展示，血压、血糖、血脂三类指标均与两类病种相关。
- 模块三的"随访记录"合并展示两类随访（通过"随访类型"字段区分）。
- 模块四的"专病档案"上下分区展示两份档案。

### 6.4 关闭条件说明

**本期结论**：**不设置关闭条件**。

- 慢病为慢性病，一旦确诊需要长期跟踪管理。
- 不像孕产妇有明确的医学终止时间（产后 42 天），慢病无合理的自动关闭时机。
- 即使患者病情稳定或长期未就诊，仍需要保留慢病空间以便后续监测。
