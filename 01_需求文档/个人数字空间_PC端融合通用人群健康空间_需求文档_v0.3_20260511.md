3. **专项人群扩展受阻**：仅做标识预留，缺少可点击入口，未来扩展路径不清

## 1. 背景与目标

### 1.1 背景

当前 PC 端个人数字空间已具备个人信息、标签统计和个人健康标签图谱能力，可从个体视角查看患者命中的健康标签、风险标签和标签类别。

下一版本需要在 PC 端将个人数字空间与人群健康空间进行融合，使健康管理人员在查看单个患者时，不仅能看到该患者的标签数据，也能查看其在人群空间下沉淀的健康特征数据，例如诊疗记录、健康史、体征指标趋势。

本需求采用"主界面保持原布局 + 人群卡片作为详情入口"的融合方式，通用人群健康空间独立为详情页，PC端一页展示全部内容，取消Tab切换。

### 1.2 建设目标

- 保留当前 PC 端个人数字空间的三区布局（个人信息/标签及人群统计卡/标签图谱）。
- 人群统计卡作为进入人群健康空间详情的唯一入口。
- 通用人群健康空间独立为详情页，PC端一页展示全量内容。
- 尽可能复用移动端通用人群健康空间既有模块和业务规则。
- 专项人群在人群卡片上做归属标识并预留详情页入口，支持未来扩展。

### 1.3 非目标

- 本期不设计专项人群（孕产妇、慢病、幼儿、重疾）详情页内容。
- 本期不改变移动端通用人群健康空间的数据口径和业务规则。

## 2. 范围与边界

### 2.1 融合对象

本期融合对象为当前患者的"通用人群健康空间"。

患者归属规则：
- 同一患者必定属于通用人群。
- 患者可能同时属于孕产妇、慢病、幼儿或重疾等专项人群。
- 本期仅落地通用人群健康空间详情页。
- 专项人群在人群卡片上做归属标识并预留详情页入口，详情页内容后续补充。

### 2.2 复用模块

PC 端复用移动端通用人群健康空间的三个模块：

| 模块   | 移动端能力                  | PC端融合要求            |
| ---- | ---------------------- | ------------------ |
| 诊疗记录 | 全部、门诊记录、住院记录；搜索筛选；详情下钻 | 复用业务规则，适配宽屏列表和详情展示 |
| 健康史  | 既往病史、家族史、过敏史、个人史       | 复用分类、搜索和空数据规则      |
| 体征数据 | 血压、血糖、BMI统计与趋势图        | 复用统计、趋势、缩放和溯源规则    |

### 2.3 不改动项

- 移动端通用人群健康空间的业务规则和数据口径。
- 现有个人健康标签图谱的能力和交互方式。
- 现有个人信息区的展示内容和布局。

## 3. 页面架构总览

### 3.1 页面层级

本次融合采用"主页（入口）→ 详情页（内容）"的两层架构：

```
个人数字空间主界面（保持原布局）
  ├─ 顶部个人信息区
  ├─ 标签及人群统计卡（改造：人群卡片作为详情入口）
  └─ 个人健康标签图谱区
       ↓ 点击人群卡片
通用人群健康空间详情页（新页面，一页平铺三模块）
  ├─ 页面头部（返回入口 / 患者基础信息 / 空间标识）
  ├─ 诊疗记录模块
  ├─ 健康史模块
  └─ 体征数据模块
```

### 3.2 跳转路径示意

```
主界面：个人数字空间
  ↓
人群统计卡展示：通用人群 / 孕产妇 / 慢病 / 幼儿 / 重疾（根据实际归属）
  ↓
点击"通用人群"卡片 → 跳转至"通用人群健康空间详情页"
点击"专项人群"卡片 → 跳转至占位页（提示"详情页后续开放"）
```

## 4. 主界面需求（个人数字空间）

### 4.1 整体说明

个人数字空间主界面**保持原有三区布局**，本期仅对"标签及人群统计卡"中的人群统计部分进行改造，其他区域保持不变。

主界面结构示意：

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部个人信息区：个人基础信息 / 数据截止时间 / 刷新             │
├────────────────────────────────────────────────────────────┤
│ 标签及人群统计卡                                              │
│   ├─ 标签统计：标签类别 / 命中标签数 / 高风险标签数             │
│   └─ 人群统计（改造）：归属人群标识 + 可点击进入详情            │
├────────────────────────────────────────────────────────────┤
│ 个人健康标签图谱区：标签图谱 / 标签类别筛选 / 图例说明          │
└────────────────────────────────────────────────────────────┘
```

### 4.2 顶部个人信息区（保留原样）

保持当前展示内容不变：

- 姓名
- 性别
- 年龄
- 民族
- 证件信息
- 联系方式
- 地址
- 数据截止时间
- 刷新入口

### 4.3 标签及人群统计卡

#### 4.3.1 标签统计部分（保留原样）

保留当前统计能力：
- 标签类别
- 命中标签数
- 高风险标签数

#### 4.3.2 人群统计部分（改造重点）

**展示规则**：

- 展示当前患者归属的**所有人群类型**。
- 归属几类人群，就在卡片上标识几类人群。
- 所有患者必定展示"通用人群"标签。
- 若患者同时归属专项人群（孕产妇 / 慢病 / 幼儿 / 重疾），追加展示对应专项人群标签（0~4个）。

**交互规则**：

- 每个人群标签均为可点击状态。
- 每个人群标签需添加**可点击标识**，明确提示可进入对应的人群健康空间详情页。

可点击标识样式建议：
- 标签右侧加箭头图标（如 "→" 或 "＞"）。
- hover 时标签高亮或显示"查看详情"文字提示。
- 鼠标悬停时光标变为"可点击"状态（pointer）。

**跳转行为**：

| 人群类型 | 点击后跳转 |
|---------|-----------|
| 通用人群 | 跳转至"通用人群健康空间详情页"（本期落地） |
| 孕产妇 | 跳转至"孕产妇健康空间详情页"占位页（提示"详情页后续开放"） |
| 慢病 | 跳转至"慢病健康空间详情页"占位页（提示"详情页后续开放"） |
| 幼儿 | 跳转至"幼儿健康空间详情页"占位页（提示"详情页后续开放"） |
| 重疾 | 跳转至"重疾健康空间详情页"占位页（提示"详情页后续开放"） |

**视觉层次建议**：

- "通用人群"作为所有患者必定归属的基础类型，视觉上可作为主标签样式。
- 专项人群作为附加标签样式，与通用人群视觉区分。
- 专项人群标签本期可采用"可点击但弱视觉强调"的样式（如浅色底、置灰等），避免用户误以为详情页已全部支持。

### 4.4 个人健康标签图谱区（保留原样）

保留当前能力，本期不做改造：

- 支持按标签类别筛选图谱。
- 支持查看可展开、不可展开、可下钻标签。
- 支持展示管理标签集可见范围。
- 图谱节点、颜色、连线和下钻逻辑不在本期重做。

## 5. 通用人群健康空间详情页（新页面）

### 5.1 页面入口与定位

**入口**：个人数字空间主界面 → 标签及人群统计卡 → 点击"通用人群"标签。

**页面定位**：独立详情页，承载当前患者的全量通用人群健康数据，PC端一页平铺展示。

### 5.2 页面头部

页面头部统一展示：

- **返回入口**：面包屑/返回按钮，支持返回个人数字空间主界面。
- **当前空间标识**：明确显示"通用人群健康空间"。
- **患者基础信息**：姓名、性别、年龄、证件信息、数据截止时间（与主界面保持一致）。

### 5.3 整体布局

**核心原则**：取消移动端Tab，三模块一页平铺展示。

**布局方案**：上一下二卡片网格布局

```text
┌───────────────────────────────────────────────────────┐
│ 页面头部：返回入口 / 患者基础信息 / 空间标识              │
├───────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────┐   │
│ │ 模块一：诊疗记录（通栏）                          │   │
│ │  子菜单/搜索/列表                                │   │
│ └─────────────────────────────────────────────────┘   │
│ ┌─────────────────────┐  ┌─────────────────────┐     │
│ │ 模块二：健康史        │  │ 模块三：体征数据      │     │
│ │  子菜单/内容展示      │  │  子菜单/统计/趋势图   │     │
│ │                      │  │                      │     │
│ └─────────────────────┘  └─────────────────────┘     │
└───────────────────────────────────────────────────────┘
```

**布局要求**：

- 诊疗记录置顶通栏展示，占据整行宽度；健康史和体征数据各占下方一半宽度。
- 上下两行等高分配，各占剩余视口高度的 50%。
- 每个卡片内部支持独立滚动，卡片间保持清晰的视觉分隔。
- 页面头部固定，卡片区域占据剩余视口高度。
- 卡片标题区固定在卡片顶部，内容区可滚动。
- 响应式适配：宽屏下上一下二网格，窄屏下可降级为单列堆叠。

### 5.4 模块一：诊疗记录

#### 功能说明

展示患者全量诊疗数据，支持按记录类型、关键词和时间范围查看。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 全部 | 展示全部门诊和住院记录 |
| 门诊记录 | 仅展示门诊相关记录 |
| 住院记录 | 仅展示住院相关记录 |

每个菜单名称后展示该菜单下的数据量。

#### 搜索筛选

- 支持关键词模糊搜索。
- 支持时间范围筛选。
- 关键词搜索和时间范围筛选可组合使用。
- 搜索筛选区位于列表上方，随内容滚动。

#### 列表展示

列表项按时间降序排列。

每条列表展示：
- 原始诊断名称
- 就诊类型：门诊 / 住院
- 就诊机构
- 就诊科室
- 就诊时间

#### 详情下钻

- 点击门诊记录，下钻展示 `诊疗详情`。
- 点击住院记录，下钻展示 `住院详情`。
- PC 端采用弹窗（Modal）形式展示详情，详情字段与移动端保持一致。

#### 三级下钻

二级详情（诊疗详情 / 住院详情）中包含以下明细类别，每类支持进一步下钻查看单条明细记录：

| 明细类型 | 入口位置     | 展示内容                                                                   |
| ---- | -------- | ---------------------------------------------------------------------- |
| 检查记录 | 门诊/住院详情内 | 检查时间、机构、科室、项目、部位、结果、建议                                                 |
| 检验记录 | 门诊/住院详情内 | 检验时间、机构、科室、项目、标本类型、结果表格（检验名称/检验值/参考值/提示）                               |
| 处方记录 | 门诊/住院详情内 | 药品名称、规格、剂量、数量、频次、服药方式                                                  |
| 手术记录 | 住院详情内    | 手术时间、机构、名称、手术信息（标志/级别/类型）、术前/术后诊断、切口愈合等级、麻醉方式                          |
| 住院详情 | 住院详情内    | 就诊时间、就诊机构、就诊科室、主诊医生、主诉、症状描述、诊断名称                                       |
| 就诊详情 | 诊疗详情内    | 入院时间、入院机构、入院科室、主治医生、病区、入院诊断、入院体征及症状住院期间主要检查结果、诊疗过程、出院时间、出院诊断、出院情况、出院医嘱 |


**交互方式**：
- 二级详情和三级明细统一使用弹窗（Modal）形式展示。
- 三级弹窗覆盖二级弹窗，弹窗头部包含"返回"按钮，点击后回到二级弹窗，保留二级弹窗的滚动位置。
- 弹窗支持关闭按钮，点击后关闭所有弹窗回到列表页。

**PC端布局**：
- 检查详情：字段列表布局，分区展示基础信息、检查结果、医生建议。
- 检验详情：表格布局，表头包含检验名称/检验值/参考值/单位/提示，异常值高亮显示。
- 处方详情：药品卡片布局，每个药品一个卡片，展示药品名称、规格、用法用量。
- 手术详情：两列网格布局，左列展示基础信息，右列展示诊断信息。

### 5.5 模块二：健康史

#### 功能说明

展示患者既往健康数据。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 既往病史 | 展示患者既往诊断记录 |
| 家族史 | 展示亲属关系及家族病史 |
| 过敏史 | 展示过敏类型和过敏源 |
| 个人史 | 展示饮食、吸烟、饮酒、运动、摄盐等生活方式信息 |

每个菜单名称后展示该菜单下的数据量。

#### 搜索规则

- `既往病史`支持关键词模糊搜索。

#### 展示规则

- 既往病史按时间降序排列。
- 既往病史列表项展示：首次确诊时间、诊断名称、诊断机构、诊断科室。
- 家族史展示亲属关系和原始诊断名称。
- 过敏史展示过敏类型和过敏源名称。
- 个人史展示饮食习惯、吸烟情况、饮酒情况、运动情况、摄盐情况。

#### 空数据规则

- 无某类亲属家族史时，不展示该亲属节点。
- 无某类过敏史时，不展示该过敏类型节点。
- 无某类个人史数据时，不展示该个人史节点。
- 当前菜单无任何数据时，显示 `暂无数据`。

### 5.6 模块三：体征数据

#### 功能说明

## 6. 专项人群预留设计

### 6.1 人群卡片归属标识规则

**标识范围**：

专项人群类型包括：
- 孕产妇
- 慢病
- 幼儿
- 重疾

**标识规则**：

- 根据当前患者的实际归属，人群卡片上动态展示归属的专项人群标签。
- 归属几类人群，就展示几类标签（可能为 0~4 个）。
- 与"通用人群"标签并列展示，视觉上做样式区分。

**样式规则**：
- 通用人群：主标签样式，视觉强调。
- 专项人群：附加标签样式，采用"浅色底+箭头"样式，提示本期详情页尚未落地。

### 6.2 预留入口行为定义

**本期行为**：

- 专项人群标签支持点击，点击后跳转至对应的专项人群健康空间占位页。
- 占位页明确提示"详情页后续开放"或"敬请期待"。
- 占位页不展示任何详情内容，避免用户误以为详情页已支持。

**占位页设计要点**：

- 展示当前空间名称（如"孕产妇健康空间"）。
- 提供返回个人数字空间主界面的入口。
- 明确提示"详情页后续开放"的文案。
