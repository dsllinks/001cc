## 1. 需求背景

随着基层医疗机构承担越来越多的慢病管理与健康干预职责，社区卫生服务机构的健康管理人员和家庭医生团队面临一个共同困境：**签约居民数量庞大，但缺乏有效的分层分类工具**，导致健康管理资源难以精准投放。

目前，管理人员通常依赖手工台账或简单筛选来识别重点人群（如慢病患者、高风险孕产妇），操作繁琐、更新滞后，且无法动态反映人群变化。

**人群健康空间**（原名：群体健康画像）是智医助理 A.I.健康管理模块下的核心功能，旨在为健康管理人员提供一套**可视化、可配置的人群分类与管理工具**，使其能够按照业务需要灵活定义人群范围，快速定位目标患者，并进入个人健康空间开展后续干预。

---

## 2. 核心目标

| 目标     | 说明                               |
| ------ | -------------------------------- |
| 灵活定义人群 | 支持多维度条件组合，满足从简单到复杂的人群划定需求        |
| 动态感知变化 | 人群人数随患者数据更新自动刷新，同比趋势可见           |
| 快速定位患者 | 在任意人群分类下，可直接查看该人群的患者名单           |
| 链接个人干预 | 从人群列表可一键跳转至个人数字健康空间，实现群体到个体的管理闭环 |
| 降低操作门槛 | 提供快速模板，减少重复配置成本，适合非技术背景的健康管理人员使用 |

## 3. 用户角色与使用场景

### 3.1 用户角色

**主要用户：社区卫生服务机构健康管理人员**

- 负责本机构签约居民的整体健康档案管理
- 需要按慢病类型、风险等级、人口学特征等维度对人群进行分类管理
- 定期向上级汇报各类重点人群的覆盖情况

**次要用户：家庭医生团队成员**

- 负责签约居民的随访、干预与健康指导
- 需要快速找到属于自己负责范围内的特定人群（如高血压未控制患者）
- 需要查看单个患者的健康档案详情以制定干预方案

### 3.2 典型使用场景

**场景一：慢病专项管理**
健康管理员小李每月需汇总辖区内高血压和糖尿病患者的数量变化。她进入人群健康空间，发现已有"慢病人群"分类，查看本月人数同比涨幅，确认患者总量，再导入随访计划。

**场景二：新建特定风险人群**
家庭医生团队正在开展心脑血管高风险专项干预。医生助手在系统中新建一个人群分类，设置条件为"心脑血管风险：高风险"，保存后系统自动统计符合条件的患者人数，团队据此开展电话随访。

**场景三：快速锁定并查看患者**
健康管理员发现某人群人数异常增加，点击"查看患者"进入患者列表，搜索特定患者姓名，点击进入其个人数字健康空间，查看近期血压记录和随访记录，判断是否需要升级干预。

**场景四：使用模板快速创建孕产妇人群**
新入职的健康管理员需要建立孕产妇人群档案，她选择"孕产妇人群"快速模板，系统自动填充"性别：女性 且 年龄：青年(18-44岁)"的条件，补充人群名称后直接保存，无需手动配置规则。

## 4. 功能详情

### 4.1 人群分类管理（列表页）

#### 功能说明

人群健康空间的主界面，展示本机构已配置的所有人群分类，是用户进行日常查看、检索和管理操作的核心入口。

#### 顶部统计看板

页面顶部呈现四项全局统计数据，帮助管理人员快速掌握整体人群管理现状：

| 统计项 | 说明 |
|--------|------|
| 人群分类总数 | 当前已配置的人群分类数量 |
| 覆盖患者总数 | 所有人群分类去重后覆盖的患者总人数 |
| 最大人群规模 | 当前人数最多的单个人群分类的人数 |
| 本月新增患者数 | 本自然月内新进入任意人群分类的患者增量 |

#### 人群列表字段

| 字段 | 说明 |
|------|------|
| 人群名称 | 用户自定义的名称 |
| 分类规则摘要 | 以可读文本形式展示该人群的核心筛选条件 |
| 状态 | 当前固定显示"已完成" |
| 备注 | 用户填写的补充说明（可为空） |
| 人群人数 | 当前符合该分类条件的患者数量，并显示与上期相比的同比涨跌幅及方向箭头 |
| 操作 | 包含"查看患者"、"编辑"、"删除"三个操作入口 |

#### 检索与排序

- 支持按**人群名称关键词**进行搜索，实时过滤列表结果
- 支持按**人群人数**排序（从多到少 / 从少到多）
- 支持按**创建时间**排序（最新创建 / 最早创建）

#### 操作说明

- **查看患者**：进入该人群下的患者名单弹窗
- **编辑**：打开侧边抽屉，加载该人群的已有配置进行修改
- **删除**：删除该人群分类（操作前需二次确认）

### 4.2 新建 / 编辑人群分类（侧边抽屉）

#### 功能说明

点击页面右上角"新建人群"或列表中某人群的"编辑"按钮后，从页面右侧滑出的操作抽屉。用户在此完成人群名称填写、快速模板选择以及规则条件的构建。

#### 基本信息字段

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 人群名称 | 必填 | 用于在列表中标识该人群，建议简洁明确 |
| 备注 | 选填 | 可填写该人群的管理目标或特殊说明 |

#### 快速模板

抽屉顶部提供四种常用人群模板，点击任意模板后，系统自动将预设条件填充到规则构建器中，用户可在此基础上进一步调整或直接保存。

| 模板名称 | 根层逻辑 | 预设条件说明 |
|----------|----------|--------------|
| 慢病人群 | 或 | 包含两个子分组：子分组1为"慢病类型：高血压"，子分组2为"慢病类型：糖尿病" |
| 幼儿人群 | 且 | 单条件"年龄：儿童(0-17岁)" |
| 孕产妇人群 | 且 | 包含两条件："性别：女性"和"年龄：青年(18-44岁)" |
| 重点疾病人群 | 或 | 包含三个子分组：子分组1"高血压风险：高风险"、子分组2"糖尿病风险：高风险"、子分组3"心脑血管风险：高风险" |

#### 多级嵌套规则构建器（核心功能）

规则构建器允许用户通过可视化方式构建多层次的人群筛选逻辑，最多支持 3 层嵌套。

**层级结构示意：**

```
第1层（根层）
├── 条件A
├── 条件B
└── 第2层（子分组）
    ├── 条件C
    ├── 条件D
    └── 第3层（嵌套分组）
        ├── 条件E
        └── 条件F
```

**层级说明：**

| 层级 | 视觉标识 | 是否可删除 | 是否可继续添加子分组 |
|------|----------|------------|----------------------|
| 第1层（根层） | 蓝色实线边框 | 不可删除 | 可添加子分组 |
| 第2层（子分组） | 浅蓝色背景 | 可删除 | 可添加嵌套分组 |
| 第3层（嵌套分组） | 更浅蓝背景 + 虚线边框 | 可删除 | 不可继续嵌套 |

**逻辑关系配置：**

每个分组可独立选择组内条件的逻辑关系：

- **且（AND）**：组内所有条件同时满足，患者才计入
- **或（OR）**：组内任一条件满足，患者即计入

**条件配置（三级联动）：**

每条筛选条件由维度 → 条件 → 值三个下拉依次确定：

| 维度分类 | 可选维度 | 示例条件值 |
|----------|----------|------------|
| 基础信息 | 年龄 | 儿童(0-17岁) / 青年(18-44岁) / 中年(45-59岁) / 老中年(60岁以上) |
| 基础信息 | 性别 | 男性 / 女性 |
| 疾病风险 | 高血压风险 | 低风险 / 中风险 / 高风险 |
| 疾病风险 | 糖尿病风险 | 低风险 / 中风险 / 高风险 |
| 疾病风险 | 心脑血管风险 | 低风险 / 中风险 / 高风险 |
| 慢病标签 | 慢病类型 | 高血压 / 糖尿病 / 冠心病 / 脑卒中 / 慢阻肺 / 恶性肿瘤 |
| 生活习惯 | 吸烟情况 | 非吸烟者 / 戒烟者 / 吸烟者 |
| 生活习惯 | 饮食习惯 | 均衡饮食 / 高盐高脂 / 素食 |

### 4.3 查看患者列表（弹窗）

#### 功能说明

在人群分类列表中点击某人群的"查看患者"后，弹出展示该人群所有患者的列表弹窗，是从人群层面下钻到个体层面的过渡界面。

#### 列表字段说明

| 字段 | 展示规则 |
|------|----------|
| 序号 | 按列表顺序自动编号 |
| 姓名 | 脱敏展示，如"张**" |
| 性别 | 男 / 女 |
| 年龄 | 实际年龄（岁） |
| 身份证号 | 脱敏展示，保留前3位和后4位，中间以"*"替代 |
| 手机号 | 依据机构授权策略展示 |
| 健康标签 | 展示该患者的主要健康标签，如"高血压""糖尿病"等 |
| 操作 | "查看个人空间"按钮 |

#### 检索与分页

- 支持按**患者姓名**关键词搜索
- 支持按**身份证号**精确搜索
- 列表采用分页方式呈现，用户可翻页浏览完整名单

---

### 4.4 查看个人数字健康空间（跳转页）

#### 功能说明

在患者列表中点击"查看个人空间"后，跳转至该患者的个人数字健康空间页面。这是从群体管理下钻到单个患者精细管理的最终落点，也是后续开展个性化健康干预的操作界面。

#### 展示内容

**基本信息**
患者姓名、性别、年龄、联系方式等基础档案信息。

**健康标签**
该患者被系统或医生打上的所有健康标签，包括慢病诊断标签、风险等级标签、生活习惯标签等。

**签约家庭医生信息**
该患者当前签约的家庭医生姓名、所在团队及签约状态。

**核心健康指标（卡片形式）**

| 指标 | 说明 |
|------|------|
| 血压 | 最近一次测量值（收缩压/舒张压），标注是否异常 |
| 血糖 | 最近一次测量值，标注是否异常 |
| BMI | 体重指数，标注体型分类 |
| 随访日期 | 最近一次随访日期及下次计划随访日期 |

**健康干预记录时间线**
以时间轴形式展示该患者历次健康干预记录（随访、生活方式指导、用药建议等），按时间倒序排列。

## 5. 业务流程

### 5.1 主流程：新建人群并查看患者

```
健康管理员进入人群健康空间（列表页）
    │
    ├─ [查看现有人群] ──→ 浏览列表，查看统计看板，了解整体人群覆盖情况
    │
    └─ [新建人群] ──→ 点击"新建人群"按钮
                            │
                            ▼
                    打开侧边抽屉
                            │
                ┌───────────┴────────────┐
                ▼                        ▼
        [选择快速模板]            [手动配置规则]
        系统自动填充预设条件       填写名称 → 配置逻辑 → 添加条件
        可在此基础上二次修改       如需复杂逻辑 → 添加子分组（最多3层）
                └───────────┬────────────┘
                            ▼
                    点击"确认保存"
                            │
                            ▼
            系统计算人群人数，人群出现在列表中
                            │
                            ▼
                点击该人群的"查看患者"
                            │
                            ▼
            弹出患者列表弹窗（可搜索、分页浏览）
                            │
                            ▼
            点击某患者的"查看个人空间"
                            │
                            ▼
            进入该患者的个人数字健康空间
                            │
                            ▼
        查看健康指标、干预记录 ──→ 决策后续干预行动
```

### 5.2 子流程：编辑已有人群

```
在列表页找到目标人群
    │
    ▼
点击"编辑"按钮
    │
    ▼
侧边抽屉打开，回显已有名称、备注和规则配置
    │
    ▼
修改条件（可增删条件、调整逻辑关系、修改分组结构）
    │
    ▼
点击"保存" ──→ 系统重新计算人群人数 ──→ 列表刷新
```

### 5.3 子流程：删除人群

```
在列表页找到目标人群
    │
    ▼
点击"删除"按钮
    │
    ▼
弹出二次确认提示："确认删除该人群分类？删除后数据不可恢复。"
    │
    ├─ [确认删除] ──→ 人群从列表中移除，统计看板同步更新
    └─ [取消]     ──→ 关闭提示，不执行任何操作
```

## 6. 交互规则

### 6.1 表单与输入规则

- 人群名称为**必填项**，保存时若为空则阻止提交，并在字段下方提示"请输入人群名称"
- 规则构建器中**至少需要配置一条有效条件**，否则不允许保存，提示"请至少添加一个筛选条件"
- 条件配置为**三级联动**（维度 → 条件 → 值），前一级未选择时后一级置灰不可操作
- 每个分组内的逻辑关系（且/或）**默认为"且"**，用户可手动切换

### 6.2 层级嵌套限制

- 第3层分组内**不显示"添加子分组"入口**，从结构上阻止超过3层的嵌套
- 根层（第1层）**不显示删除按钮**，仅第2层和第3层的分组可被删除
- 删除某一分组时，该分组下的**所有条件和嵌套子分组一并删除**，删除前弹出确认提示

### 6.3 快速模板使用规则

- 选择模板时，若规则构建器中**已有用户自行配置的内容**，系统弹出提示："选择模板将覆盖当前已配置的规则，是否继续？"，用户确认后方可覆盖
- 模板填充的条件可以在构建器中**自由增减和修改**，模板本身不锁定任何字段

### 6.4 操作反馈规则

| 操作 | 成功反馈 | 失败反馈 |
|------|----------|----------|
| 保存人群 | 抽屉关闭，列表刷新，顶部提示"保存成功" | 抽屉保持打开，提示具体错误原因 |
| 删除人群 | 列表移除该条目，顶部提示"删除成功" | 提示"删除失败，请重试" |
| 搜索患者 | 列表实时过滤展示匹配结果 | 无匹配结果时显示"未找到符合条件的患者" |

### 6.5 数据展示边界情况

- 某人群**人数为 0** 时，仍正常显示该人群分类，人数显示"0 人"，同比数据显示"—"
- **上期无数据**时同比栏显示"—"，不展示涨跌箭头
- 患者列表中健康标签为空时，该列显示"—"
- 个人健康空间中某项指标**无记录**时，对应卡片显示"暂无数据"
- 健康干预记录时间线**无任何记录**时，显示"暂无干预记录"

### 6.6 隐私与脱敏规则

- **患者姓名**：展示姓氏，名字部分用"*"替代（如"张**"）
- **身份证号**：保留前3位和后4位，中间以连续"*"替代
- **手机号**：展示策略依据机构数据授权策略执行，默认完整展示

## 7. 数据需求

### 7.1 人群分类数据字段

| 字段 | 说明 |
|------|------|
| 人群名称 | 用户自定义，用于标识该人群 |
| 备注 | 选填，管理说明 |
| 状态 | 固定为"已完成" |
| 规则配置 | 多层嵌套的条件组合（逻辑关系 + 条件列表 + 子分组结构） |
| 人群人数（当前） | 当前符合规则的患者数量 |
| 人群人数（上期） | 用于计算同比涨跌，对应上一统计周期 |
| 创建时间 | 人群分类首次保存的时间 |
| 最近更新时间 | 人群规则最近一次修改的时间 |

### 7.2 条件维度数据依赖

规则构建器的条件选项来源于患者健康档案的不同数据模块，需与各模块数据保持同步更新：

| 维度 | 数据来源 |
|------|----------|
| 年龄、性别 | 患者基础档案 |
| 高血压/糖尿病/心脑血管风险 | A.I.健康风险评估结果 |
| 慢病类型 | 慢病管理标签 |
| 吸烟情况、饮食习惯 | 生活习惯问卷记录 |

### 7.3 患者列表展示字段

| 字段 | 来源 | 备注 |
|------|------|------|
| 姓名 | 患者基础档案 | 脱敏展示 |
| 性别 | 患者基础档案 | — |
| 年龄 | 患者基础档案 | 实际周岁 |
| 身份证号 | 患者基础档案 | 脱敏展示 |
| 手机号 | 患者基础档案 | 依授权策略展示 |
| 健康标签 | 慢病管理 + 风险评估 | 汇总该患者的核心标签 |

### 7.4 个人数字健康空间展示数据

| 板块 | 所需字段 | 数据来源 |
|------|----------|----------|
| 基本信息 | 姓名、性别、年龄、联系方式 | 患者基础档案 |
| 健康标签 | 所有标签及打标时间 | 慢病管理 + 风险评估 |
| 签约医生 | 医生姓名、团队、签约状态 | 家庭医生签约数据 |
| 血压 | 最近测量值（收缩压/舒张压）、测量时间 | 健康指标记录 |
| 血糖 | 最近测量值、测量时间 | 健康指标记录 |
| BMI | 最近计算值、体型分类 | 健康指标记录 |
| 随访日期 | 上次随访日期、下次计划日期 | 随访管理数据 |
| 干预时间线 | 干预类型、日期、内容摘要、操作人 | 健康干预记录 |
