## 1. 背景与目标

### 1.1 背景

通用人群健康空间PC端融合需求（v0.3）已完成定稿，主界面三区布局及人群卡片入口机制已确定。当前孕产妇人群在主界面人群卡片中作为专项人群标签存在，点击后跳转至占位页（提示"详情页后续开放"）。

本期需求承接 v0.3 的预留扩展机制，正式落地"孕产妇健康空间"详情页：

- 移动端孕产妇健康空间已上线，业务规则成熟。
- V1.1 版本规划将"孕产妇人群"作为重点人群之一，需要在 PC 端为健康管理人员提供完整的孕产妇视图。
- 主界面布局保持不变，仅本期将孕产妇标签的跳转目标从"占位页"切换为"孕产妇健康空间详情页"。

### 1.2 建设目标

- 复用主界面三区布局，**主界面不做任何改动**。
- 落地"孕产妇健康空间详情页"，承载当前患者的孕产期全量健康数据。
- 复用通用人群详情页的左侧锚点导航 + 右侧分区内容的整体框架。
- 复用移动端孕产妇健康空间的业务规则（孕周计算、孕期判断、血糖异常范围、体重增长曲线等）。
- 围绕健康管理人员视角组织模块顺序，强调诊疗记录、核心指标和风险预警。
- 支持产前、产后两种状态的差异化展示。

### 1.3 非目标

- 本期不涉及主界面布局调整。
- 本期不重做移动端孕产妇健康空间业务规则和数据口径。
- 本期不实现"科普知识"模块（运营成本高，PC健康管理视角无强需求）。
- 本期不实现"孕检安排（待办事项）"模块（属于居民自服务交互，PC健康管理人员场景不适用）。
- 本期不实现"胎儿发育状况"模块（依赖B超结构化数据，数据来源待确认，建议后续版本补充）。

## 2. 范围与边界

### 2.1 融合对象

本期融合对象为当前患者的"孕产妇健康空间"。

患者归属规则：
- 仅满足展示条件（见第6章）的患者，孕产妇标签才会出现在主界面人群卡片中。
- 满足关闭条件时，孕产妇标签自动从人群卡片中移除。
- 孕产妇空间在"产前"和"产后"两种状态下，展示内容存在差异。

### 2.2 复用模块

PC 端复用移动端孕产妇健康空间的核心模块：

| 模块 | 移动端能力 | PC端融合要求 |
|------|----------|------------|
| 孕产健康视图 | 按孕期阶段分Tab展示诊疗与随访记录；搜索筛选；详情下钻 | 复用业务规则，子菜单适配PC宽屏列表 |
| 孕期核心指标 | 血压、血糖（孕妇专用范围）、体重（孕前BMI增长曲线）；统计、趋势图、溯源 | 复用业务规则，趋势图宽屏适配 |
| 孕期风险预警 | 风险指标解读：指标名称、风险等级、评估依据、风险提示、干预建议 | 复用业务规则，PC端宽屏表格化展示 |

PC 端新增模块：

| 模块 | 移动端是否有 | PC端新增说明 |
|------|------------|------------|
| 产检记录 | 移动端无独立模块 | PC端宽屏适合时间线展示历次产检关键结果 |
| 孕期AI摘要与个性化建议 | 移动端无 | 落地需求框架规划的孕期AI摘要能力（依赖讯飞医疗大模型） |

### 2.3 不改动项

- 个人数字空间主界面布局和交互（沿用 v0.3 三区布局）。
- 主界面人群卡片的展示规则、可点击标识规则。
- 移动端孕产妇健康空间的业务规则、数据口径、判断逻辑。
- 通用人群健康空间详情页的设计（独立详情页，不受本期影响）。

### 2.4 本期不做项

- 科普知识模块。
- 孕检安排（待办事项）模块。
- 胎儿发育状况模块（双顶径、股骨长、腹围等）。
- 婴幼儿、慢病、重疾等其他专项人群详情页。

## 5. 孕产妇健康空间详情页

### 5.1 页面入口与定位

**入口**：个人数字空间主界面 → 标签及人群统计卡 → 点击"孕产妇"标签。

**页面定位**：独立详情页，承载当前患者孕产期的全量健康数据，支持产前、产后两种状态的差异化展示。

**目标用户视角**：健康管理人员（医生/管理员）视角下查看患者孕产期健康状态，强调"问题导向、风险可见、数据可溯"。

### 5.2 页面头部（含孕周进度区）

孕产妇详情页头部相比通用人群详情页头部，**新增"孕周进度区"**，整体结构为三层：

```text
┌──────────────────────────────────────────────────────────────┐
│ 第1层：返回入口  |  孕产妇健康空间                              │
├──────────────────────────────────────────────────────────────┤
│ 第2层：姓名 性别 年龄 证件号  |  数据截止时间                    │
├──────────────────────────────────────────────────────────────┤
│ 第3层：[孕中期]  第 24+3 周   预产期：2026-09-15（距今 127 天） │
│        ●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━○ │
│        孕早期    孕中期▲                    孕晚期       预产期 │
└──────────────────────────────────────────────────────────────┘
```

#### 第1层：导航条

- 返回入口：面包屑或返回按钮，支持返回个人数字空间主界面。
- 当前空间标识：明确显示"孕产妇健康空间"。

#### 第2层：患者基础信息

- 姓名、性别、年龄、证件号、数据截止时间。
- 字段口径与通用人群详情页保持一致。

#### 第3层：孕周进度区（孕产妇专属）

**产前状态（孕早/中/晚期）**：

- 左侧标签：孕期阶段（孕早期 / 孕中期 / 孕晚期）。
- 中间文字：当前孕周（格式"周+天"，如 24+3 周）。
- 右侧文字：预产期 + 距今倒计时天数。
- 下方进度条：直观呈现当前所处阶段，PC端宽屏完整展示四阶段（孕早期 / 孕中期 / 孕晚期 / 预产期）。

**产后状态（产褥期）**：

- 左侧标签：产褥期。
- 中间文字：分娩日期（若分娩日期获取不到，展示"分娩日期 ---"）。
- 右侧文字：产后第 N 天 + 距空间关闭剩余天数。
- 下方进度条：替换为"分娩→产后42天"的产后进度条，展示当前所处天数位置。

#### 头部固定行为

- 第1层和第2层固定，不随页面滚动消失。
- 第3层孕周进度区固定（强调"孕期阶段"是孕产妇详情页的核心上下文）。

### 5.3 整体布局

**布局模式**：沿用通用人群详情页的"左侧锚点导航 + 右侧分区内容"框架。

**核心原则**：取消Tab，多模块一页平铺展示，左侧锚点导航支持快速定位。

```text
┌─────────────────────────────────────────────────────────────┐
│ 页面头部（含孕周进度区）                                       │
├─────────────┬───────────────────────────────────────────────┤
│             │ 模块一：孕产健康视图                            │
│ 锚点导航     │   子菜单（按孕期阶段）/ 搜索筛选 / 列表          │
│             │                                                │
│ 孕产健康视图 │ 模块二：孕期核心指标                            │
│ 孕期核心指标 │   子菜单（血压/血糖/体重）/ 统计 / 趋势图        │
│ 孕期风险预警 │                                                │
│ 产检记录     │ 模块三：孕期风险预警                            │
│ 孕期AI摘要   │   风险指标列表 / 风险等级 / 干预建议            │
│             │                                                │
│             │ 模块四：产检记录                                │
│             │   时间线 / 关键检查项目                         │
│             │                                                │
│             │ 模块五：孕期AI摘要与个性化建议                  │
│             │   AI摘要 / 饮食 / 运动 / 作息建议               │
└─────────────┴───────────────────────────────────────────────┘
```

**布局要求**：

- 左侧锚点导航固定，滚动时高亮当前模块对应的锚点。
- 右侧内容区垂直滚动，五模块依次展示。
- 点击左侧锚点可快速跳转至对应模块。
- 模块间保持清晰的分隔（标题、分割线或留白）。
- 进入详情页默认高亮"孕产健康视图"模块。

### 5.4 模块一：孕产健康视图（诊疗记录）

#### 功能说明

展示患者孕产期间全量的诊疗与随访数据，支持按孕期阶段、关键词和时间范围查看。

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 全部 | 展示孕产期间全部诊疗记录和公卫随访记录 |
| 孕早期 | 仅展示孕早期（第1-13+6周）的记录 |
| 孕中期 | 仅展示孕中期（第14-27+6周）的记录 |
| 孕晚期 | 仅展示孕晚期（第28周以后）的记录 |
| 分娩期 | 仅展示分娩相关记录 |
| 产褥期 | 仅展示产褥期（产后42天内）的记录 |

每个菜单名称后展示该菜单下的数据量。

孕期阶段归属规则：每条记录按"就诊时间 / 随访时间"映射到对应孕期阶段。归属规则完全复用移动端。

#### 搜索筛选

- 支持关键词模糊搜索：项目名称（含公卫随访项目名称、就诊记录诊断名称）、机构、科室。
- 支持时间范围筛选。
- 关键词搜索和时间范围筛选可组合使用。
- 搜索筛选区位于列表上方，随内容滚动（沿用 v0.3 处理）。

#### 列表展示

列表项按时间降序排列（随访时间和就诊时间统一参与排序）。

每条列表展示：
- 原始诊断名称 / 公卫随访项目名称
- 类型：随访 / 门诊 / 住院
- 机构
- 科室
- 时间

#### 详情下钻

- 点击门诊记录，下钻展示"诊疗详情"。
- 点击住院记录，下钻展示"住院详情"。
- 点击公卫随访记录，下钻展示"随访详情"。
- **诊疗详情和住院详情的字段与通用人群诊疗记录详情完全一致**（移动端已明确）。
- PC 端展示方式（弹窗 / 抽屉 / 详情区）见 7.1，建议与通用人群详情页保持一致。

#### 空数据规则

- 当前菜单无任何数据时，显示"暂无数据"。

### 5.5 模块二：孕期核心指标（体征数据）

#### 功能说明

展示患者孕产期间核心体征指标的统计信息和趋势变化，包含血压、血糖、体重三项。

#### 与通用人群体征数据的差异

| 维度 | 通用人群体征数据 | 孕产妇孕期核心指标 |
|------|---------------|------------------|
| 指标项 | 血压、血糖、BMI | 血压、血糖、**体重**（替换BMI） |
| 血糖异常范围 | 成人通用范围 | 孕妇专用范围 |
| 体重/BMI | BMI统计与趋势 | 体重增长曲线（基于孕前BMI推荐区间） |
| 统计时间范围 | 历年 / 近一年 | **本次孕产期间** |
| 趋势图参考线 | 无参考线 | 体重趋势图叠加推荐增长范围区间 |
| 溯源 | 支持 | 支持（复用通用人群溯源弹窗） |

#### 子菜单

| 菜单 | 说明 |
|------|------|
| 血压 | 展示孕产期间血压统计和趋势 |
| 血糖 | 展示孕产期间血糖统计和趋势（孕妇专用异常范围） |
| 体重 | 展示孕产期间体重指标提示和趋势（基于孕前BMI推荐区间） |

默认选中"血压"。

#### 统计信息

每个指标下展示：
- 孕产期间检查次数
- 孕产期间异常次数

> 与通用人群体征数据的"历年 / 近一年"统计口径不同：孕产妇统计范围限定在**本次孕产期间**（从末次月经时间起算，至空间关闭日止）。

#### 血压

- 趋势图：横坐标为日期，纵坐标为血压数值（收缩压 / 舒张压）。
- 支持溯源：点击数据点弹出溯源弹窗。

#### 血糖

- 趋势图：横坐标为日期，纵坐标为血糖数值。
- 异常判断规则（**孕妇专用，沿用移动端**）：
  - 空腹血糖 ≥ 5.1 mmol/L 为异常。
  - 餐后1小时 ≥ 10.0 mmol/L 为异常。
  - 餐后2小时 ≥ 8.5 mmol/L 为异常。
- 若餐后血糖取不到时间，默认为餐后2小时数值。
- 同一天的异常次数最多算一次（同一天空腹与餐后均异常按1次计算）。
- 支持溯源。

#### 体重

- 文案规则（沿用移动端）：根据孕前BMI、孕期阶段动态生成提示文案。
  - 孕早期文案模板：`您的孕前 BMI 为 XXX，处于XXX范围。依据《妊娠期妇女体重增长推荐值标准》指南建议，您整个孕期体重增长需控制在XXXX kg范围内。鉴于您目前处于孕早期，建议本阶段体重增长控制在 XXX kg`
  - 孕中/晚期文案模板：`您的孕前 BMI 为 XXX，处于XXX范围。依据《妊娠期妇女体重增长推荐值标准》指南建议，您整个孕期体重增长需控制在XXXX kg范围内。鉴于您目前处于孕X期，建议本阶段体重每周增长 XXX kg（0.26~0.48 kg）为宜`
- 趋势图：横坐标为日期，纵坐标为体重。**叠加推荐增长区间的上下限参考线**。
- 具体推荐区间数值（按孕前BMI分类）：沿用移动端附件《妊娠期妇女体重增长推荐值标准》表单。
- 支持溯源。

#### 溯源弹窗

- 点击趋势图数据点后，弹窗展示该数据点的溯源信息。
- 溯源字段与健康画像PC端既有溯源字段保持一致。
- 弹窗支持关闭后回到当前指标趋势图。

### 5.6 模块三：孕期风险预警

#### 功能说明

展示患者孕产期间命中的风险指标，按风险等级和评估依据呈现，辅以风险提示和干预建议。

#### 风险指标字段

每条风险指标展示：
- **指标名称**：如妊娠高血压、妊娠糖尿病等。
- **风险等级**：低 / 中 / 高（视觉上使用色块或图标区分）。
- **评估依据**：触发该风险等级的具体指标值和判断逻辑。
- **风险提示**：该风险的医学说明文字。
- **干预建议**：针对该风险的建议措施。

#### 孕期阶段对应规则

不同孕期阶段对应的风险指标集不同，**沿用移动端附件《孕产妇健康知识》**：
- 孕早期风险指标集：见《孕早期个性化风险分析》表单。
- 孕中期风险指标集：见《孕中期个性化风险分析》表单。
- 孕晚期风险指标集：见《孕晚期个性化风险分析》表单。
- 产褥期风险指标集：待医疗专家确认（见 Q5 待确认）。

#### 展示规则

- 按"风险等级"降序展示（高 > 中 > 低）。
- 同等级内按"指标名称"或"触发时间"排序（待原型评审确认）。
- 无命中风险时显示"暂无风险预警"。

#### PC端展示形式

PC 端宽屏适合采用"列表+展开详情"或"卡片网格"形式，每条风险占据较宽空间展示完整信息（不像移动端折叠）。具体形式见原型评审。

### 5.7 模块四：产检记录

#### 功能说明

展示患者历次产检的关键检查项目结果，按时间线形式呈现。本模块为PC端新增。

#### 字段

每次产检记录展示：
- 产检时间
- 产检孕周（自动计算：产检时间 - LMP）
- 产检机构
- 关键检查项目结果（B超、血常规、唐筛、糖耐量等）
- 产检结论（若有）

#### 展示形式

- 时间线形式：从最早一次产检到最近一次产检，按时间正序排列；或时间降序排列（待原型评审确认）。
- 每条产检记录以卡片形式呈现，卡片头部为产检时间和产检孕周，卡片内容为关键检查项目。

#### 空数据规则

- 无产检记录时显示"暂无产检记录"。

#### 数据依赖说明

- 产检数据来源：HIS就诊记录中的产检相关项目、公卫随访中的产前检查记录。
- 关键检查项目的结构化提取：依赖现有标签体系和指标体系的能力（具体见 Q2 待确认）。
- 若关键检查项目无法结构化提取，本期可降级为"展示产检时间+机构+原始诊断名称"，不展示具体检查值。

### 5.8 模块五：孕期AI摘要与个性化建议

#### 功能说明

基于讯飞医疗大模型，结合患者孕产期数据生成阶段性健康摘要和个性化建议。本模块为PC端新增。

#### 展示内容

**孕期AI摘要**：
- 基于当前孕期阶段和健康数据，生成本阶段健康状况总结。
- 通俗易懂的表达，符合居民端"AI健康管家"的语言风格。
- 展示生成时间戳。

**个性化建议**：
- **饮食建议**：针对当前孕周的饮食建议。
- **运动建议**：针对当前孕周的运动建议。
- **作息建议**：针对当前孕周的作息建议。

#### 生成策略

- 生成时机：定时生成（具体频率待技术确认）或按需生成。
- 生成依据：患者孕产期事实数据（HIS、公卫平台等）+ 已生成的标签数据。
- 输出形式：结构化文本，分摘要、饮食、运动、作息四个板块展示。

#### 本期落地范围说明

孕期AI摘要本期是否落地，依赖讯飞大模型在孕产妇场景下的接入进度。本模块在文档中作为完整需求设计，实际是否进入本期开发由技术评估后决定（见 Q4 待确认）。

若本期不落地，本模块可降级为"占位区"，提示"AI摘要功能开发中"。

### 5.9 产后状态特殊展示规则

患者从产前状态切换为产褥期后，详情页各模块展示存在以下差异：

| 区域/模块 | 产前状态（孕早/中/晚期） | 产褥期状态 |
|----------|--------------------|----------|
| 头部进度区 | 孕期阶段标签 + 孕周 + 预产期倒计时 + 孕期进度条 | 产褥期标签 + 分娩日期 + 产后第N天 + 距空间关闭剩余天数 + 产后进度条 |

| 孕期核心指标 | 血压/血糖/体重（孕期范围、孕期文案） | 血压/血糖/体重（产后恢复视角，体重趋势图参考线切换为产后恢复推荐区间） |

## 6. 展示条件与关闭条件

### 6.1 展示条件（孕产妇标签出现的前提）

当前患者满足以下**任一**条件时，主界面人群卡片展示"孕产妇"标签：

1. 居民存在"孕妇专项档案"记录，且"末次月经时间"不为空。
2. 居民存在"第一次产前检查服务记录"，且"末次月经时间"不为空。

**末次月经时间为空时，不展示孕产妇标签**（无法计算孕周）。

### 6.2 孕期阶段判断规则

基于"末次月经时间（LMP）"和分娩记录判断当前孕期阶段：

| 孕期阶段 | 判断规则 |
|---------|---------|
| 孕早期 | 第 1 周 ~ 第 13+6 周（未获取到分娩记录） |
| 孕中期 | 第 14 周 ~ 第 27+6 周（未获取到分娩记录） |
| 孕晚期 | 第 28 周以后（未获取到分娩记录） |
| 产褥期 | 已获取到分娩记录后，至产后 42 天结束 |

#### 孕周计算

- 当前孕周 = 当前日期 - 末次月经日期（LMP），格式为"周+天"。
- 例如：第 13+6 周表示 13 周 + 6 天。

#### 预产期计算

- 优先取"预产期"字段。
- 若"预产期"为空，按 LMP + 280 天计算。

#### 异常分娩处理

- 若**一直获取不到分娩记录**，从末次月经时间后推 **301 天**自动进入产褥期，产褥期后 42 天自动关闭空间。

### 6.3 关闭条件（孕产妇标签自动消失）

满足以下**任一**条件时，孕产妇空间关闭，主界面人群卡片不再展示孕产妇标签：

1. **流产相关诊断**：居民出现流产相关诊断（具体诊断明细沿用移动端附件）。
2. **产后42天**：进入产褥期后，产后第 42 天当天结束，孕产妇空间自动关闭。

### 6.4 空间关闭后的入口处理

孕产妇空间关闭后：
- 主界面人群卡片不展示孕产妇标签，无跳转入口。
- 历史孕产期数据的归档与查询入口：见 Q8 待确认。

- [ ] 血糖异常判断使用孕妇专用范围（空腹≥5.1/餐后1h≥10.0/餐后2h≥8.5）。
- [ ] 同一天血糖异常最多计1次。
- [ ] 体重趋势图叠加推荐增长区间上下限参考线。
- [ ] 体重文案按孕前BMI和当前孕期阶段动态生成。
- [ ] 点击趋势图数据点弹出溯源弹窗。
- [ ] 溯源字段与通用人群体征数据保持一致。

### 9.6 孕期风险预警模块

- [ ] 展示当前孕期阶段对应的风险指标集。
- [ ] 每条风险展示指标名称、风险等级、评估依据、风险提示、干预建议。
- [ ] 风险等级有视觉区分（色块/图标）。
- [ ] 无命中风险时显示"暂无风险预警"。

### 9.7 产检记录模块

- [ ] 时间线形式展示历次产检记录。
- [ ] 每条产检展示时间、孕周、机构、关键检查项目、结论。
- [ ] 无产检记录时显示"暂无产检记录"。
- [ ] 关键检查项目降级方案：无法结构化提取时展示"产检时间+机构+原始诊断名称"。

### 9.8 孕期AI摘要模块

- [ ] 展示AI摘要、饮食建议、运动建议、作息建议四个板块。
- [ ] 展示生成时间戳。
- [ ] 本期不落地时显示"AI摘要功能开发中"占位提示。

### 9.9 产后状态特殊展示

- [ ] 产褥期状态下，头部进度区切换为产后视图。
- [ ] 产褥期状态下，孕产健康视图默认选中"产褥期"Tab。
- [ ] 产褥期状态下，体重趋势图参考线切换为产后恢复推荐区间。
- [ ] 产褥期状态下，风险预警展示产后风险指标集（待确认）。

### 9.10 展示/关闭条件

- [ ] 末次月经时间为空时，孕产妇标签不展示。
- [ ] 出现流产相关诊断时，孕产妇空间关闭，标签从主界面移除。
- [ ] 产后42天当天自动关闭空间。
- [ ] 一直获取不到分娩记录时，LMP后推301天自动进入产褥期。

- [ ] 移动端业务规则在PC端保持一致。
- [ ] PC端布局适配宽屏展示。

## 11. 实施优先级建议

**P0（核心，必须本期落地）**：
- 页面头部含孕周进度区
- 孕产健康视图（诊疗记录）
- 孕期核心指标（血压/血糖/体重）
- 展示条件与关闭条件

**P1（重要，本期落地）**：
- 孕期风险预警
- 产检记录（含降级方案）

**P2（增值，视资源决定）**：
- 孕期AI摘要与个性化建议

**暂缓（后续版本）**：
- 胎儿发育状况（双顶径、股骨长等，依赖B超结构化数据）
- 科普知识
- 孕检安排待办事项
