## 一、需求背景

### 1.1 项目背景

依托星火大模型，紧扣国家"4128"数智健康体系，构建全生命周期 AI 数字健康画像，实现"便民、助医、辅政、促研"，打造智慧健康生态，推动城市医疗健康数智化转型。

### 1.2 优化背景

**原模块名称**：群体健康画像  
**新模块名称**：人群健康空间

**当前问题**：
1. 人群分类配置流程复杂，三级联动选择器操作路径长
2. 分组类型与标签概念混淆，用户理解成本高
3. 列表信息展示维度不足，缺少关键数据可视化
4. 缺少从"群体分析"到"个体明细"的完整链路
5. 空状态引导不足，新用户上手困难

### 1.3 业务价值

- **辅政端**：通过精准人群分类，支撑政府健康政策决策，实现群体健康监测与趋势预测
- **助医端**：帮助医生快速定位重点关注人群，提升慢病管理和预防干预效率
- **促研端**：为临床科研提供标准化人群样本筛选工具

## 二、目标用户

### 2.1 主要用户

| 用户角色 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| **卫健委管理人员** | 区域健康监管、政策制定 | 快速了解重点人群分布与健康趋势 |
| **公卫医生** | 慢病管理、健康干预 | 精准筛选目标人群，制定干预方案 |
| **临床医生** | 患者管理、科研分析 | 按疾病/风险快速定位患者群体 |
| **数据分析师** | 健康数据分析 | 灵活配置人群规则，导出分析数据 |

### 2.2 用户画像

- 熟悉医疗健康业务，但技术能力参差不齐
- 需要快速、直观的操作体验
- 关注数据准确性和实时性
- 需要从宏观到微观的分析能力

## 三、核心功能需求

### 3.1 功能架构

```
人群健康空间
├── 人群分类管理
│   ├── 新建人群分组（核心功能）
│   ├── 编辑人群分组
│   ├── 删除人群分组
│   └── 复制人群分组
├── 人群列表展示
│   ├── 列表视图
│   ├── 筛选与排序
│   └── 搜索功能
├── 人群详情分析
│   ├── 人群概览
│   ├── 健康特征分析
│   └── 患者明细列表
└── 数据导出
    ├── 导出人群列表
    └── 导出患者明细
```

### 3.2 功能详细说明

#### 3.2.1 新建人群分组（P0 - 核心功能）

**功能描述**：支持用户通过手动配置标签规则，定义特定人群分类。

**功能要点**：

1. **创建方式选择**
   - 快速创建：基于预置模板快速生成
   - 自定义创建：灵活配置标签组合规则

2. **标签组合器（替代原三级联动）**
   - 支持多维度标签选择：
     - 疾病风险（高血压、糖尿病、冠心病等）
     - 年龄段（0-6岁、7-17岁、18-59岁、60岁以上等）
     - 性别（男性、女性）
     - 地区（区县、街道、社区）
     - 风险等级（高危、中危、低危）
     - 健康状态（健康、亚健康、患病）
   - 支持逻辑组合：
     - "且"关系：同时满足多个条件
     - "或"关系：满足任一条件
   - 实时预览人群数量

3. **预置人群模板**
   - 慢病人群：高血压人群、糖尿病人群、冠心病人群等
   - 重点人群：孕产妇、0-6岁儿童、65岁以上老人
   - 风险人群：高危人群、中危人群
   - 年龄段人群：学龄前儿童、学龄期儿童、中老年人

4. **基本信息配置**
   - 人群名称（必填，50字以内）
   - 备注描述（选填，200字以内）
   - 分组规则（自动生成标签展示）

**交互流程**：

```
步骤1：点击"新建分组"按钮
  ↓
步骤2：选择创建方式
  ├─ 快速创建 → 选择模板 → 跳转步骤4
  └─ 自定义创建 → 进入步骤3
  ↓
步骤3：配置分组规则
  ├─ 添加标签条件
  ├─ 设置逻辑关系（且/或）
  └─ 实时查看预览人群数量
  ↓
步骤4：填写基本信息
  ├─ 输入人群名称
  └─ 输入备注描述（可选）
  ↓
步骤5：确认创建
  ├─ 系统计算人群数据
  └─ 返回列表页，显示新建人群
```

**字段说明**：

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|---------|---------|---------|------|
| 人群名称 | 文本 | 是 | 最多50字，不可重复 |
| 分组规则 | 标签组合 | 是 | 至少选择1个标签 |
| 备注描述 | 文本 | 否 | 最多200字 |
| 创建时间 | 时间戳 | 自动 | 系统自动记录 |
| 更新时间 | 时间戳 | 自动 | 规则变更时更新 |

**异常处理**：

- 人群名称重复：提示"该人群名称已存在，请修改"
- 未选择标签：提示"请至少选择一个分组条件"
- 人群数量为0：提示"当前规则未匹配到任何人群，请调整条件"

#### 3.2.2 人群列表展示（P0）

**功能描述**：以列表形式展示所有已创建的人群分组，支持查看、编辑、删除等操作。

**列表字段**：

| 字段名称 | 展示形式 | 说明 |
|---------|---------|------|
| 序号 | 数字 | 自动编号 |
| 人群名称 | 文本 | 支持点击进入详情页 |
| 分组规则 | 标签组 | 以标签形式展示（如：`高血压` `60-70岁` `女性`） |
| 人群数量 | 数字+趋势图标 | 突出显示，支持点击查看趋势 |
| 更新时间 | 时间 | 显示最后计算时间 |
| 状态 | 状态标识 | 已完成/计算中/已过期 |
| 操作 | 按钮组 | 查看/编辑/复制/删除 |

**功能要点**：

1. **筛选功能**
   - 按分组规则类型筛选（疾病、年龄、风险等）
   - 按状态筛选（全部/已完成/计算中）
   - 按时间范围筛选

2. **排序功能**
   - 按人群数量排序（升序/降序）
   - 按更新时间排序（最新/最早）
   - 按创建时间排序

3. **搜索功能**
   - 支持按人群名称模糊搜索
   - 支持按标签关键词搜索

4. **批量操作**
   - 批量删除
   - 批量导出

5. **空状态优化**
   - 无数据时显示引导图
   - 提示文案："暂无人群分组，点击右上角「新建分组」开始创建"
   - 提供快速创建入口

**状态说明**：

| 状态 | 颜色标识 | 说明 |
|------|---------|------|
| 已完成 | 绿色 | 人群数据计算完成，可正常查看 |
| 计算中 | 橙色 | 后台正在计算人群数据 |
| 已过期 | 灰色 | 规则配置已过期，需重新计算 |

---

#### 3.2.3 人群详情分析（P1 - 新增功能）

**功能描述**：点击人群名称或"查看"按钮，进入人群详情页，展示该人群的整体画像与患者明细。

**页面结构**：

```
人群详情页
├── 顶部：面包屑导航 + 人群名称
├── 区域1：人群概览卡片
│   ├── 总人数 + 环比趋势
│   ├── 性别分布（饼图）
│   ├── 年龄分布（柱状图）
│   └── 地区分布（地图/列表）
├── 区域2：健康特征分析
│   ├── 主要疾病分布（TOP5）
│   ├── 风险等级分布
│   └── AI生成的人群洞察（文本）
└── 区域3：患者明细列表
    ├── 筛选：姓名/身份证号
    ├── 列表字段：姓名/性别/年龄/身份证号/手机号/标签
    └── 操作：查看个体标签（跳转个人数字健康空间）
```

**功能要点**：

1. **人群概览**
   - 总人数：显示当前人群总数，对比上周/上月变化趋势
   - 性别分布：饼图展示男女比例
   - 年龄分布：柱状图展示各年龄段分布
   - 地区分布：按区县/街道展示人群分布

2. **健康特征分析**
   - 主要疾病分布：展示TOP5疾病及占比
   - 风险等级分布：高危/中危/低危人数及占比
   - AI人群洞察：基于星火大模型生成的人群特征描述

3. **患者明细列表**
   - 支持按姓名、身份证号搜索
   - 点击"查看个体标签"跳转到"个人数字健康空间"
   - 支持导出患者明细（Excel格式）

---

#### 3.2.4 其他功能

**编辑人群分组**：修改人群名称、备注、分组规则，修改后自动重新计算

**删除人群分组**：支持单个删除和批量删除，二次确认防误删

**复制人群分组**：基于已有人群快速创建相似人群

**数据导出**：支持导出人群列表和患者明细（Excel格式）

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 列表加载时间 | ≤2秒 |
| 人群数据计算时间 | ≤10秒（万人以下）；≤30秒（万人以上） |
| 详情页加载时间 | ≤3秒 |
| 导出响应时间 | ≤5秒（万条以下数据） |

### 4.2 兼容性要求

- 浏览器：Chrome 90+、Edge 90+、Firefox 88+
- 分辨率：1920×1080及以上
- 支持响应式布局（可选）

### 4.3 安全性要求

- 患者隐私数据脱敏展示（身份证号、手机号部分隐藏）
- 操作日志记录（创建、编辑、删除操作）
- 权限控制：按角色分配功能权限

### 4.4 可用性要求

- 界面简洁直观，核心操作不超过3步
- 关键操作提供二次确认
- 异常情况提供明确的错误提示

## 五、交互设计要点

### 5.1 视觉设计原则

- **数据可视化优先**：人群数量、趋势等关键数据用图表展示
- **层级清晰**：主次信息分明，重点数据突出显示
- **状态可识别**：用颜色区分不同状态（绿/橙/灰）
- **操作可预期**：按钮文案清晰，操作结果可预见

### 5.2 关键交互说明

#### 5.2.1 标签组合器交互

```
[添加条件] 按钮
  ↓
弹出标签选择面板
  ├─ 左侧：标签分类（疾病/年龄/性别/地区/风险）
  └─ 右侧：具体标签列表（支持搜索）
  ↓
选择标签后自动添加到规则区
  ↓
规则区显示：
  [高血压] [且] [60-70岁] [且] [女性]
  ↓
实时显示预览人群数量：
  "预计匹配 1,234 人"
```

#### 5.2.2 人群数量展示交互

- 列表页：数字 + 小趋势图标（↑/↓）
- 点击数字：弹出趋势图（近7天/近30天人群数量变化）
- 悬浮提示：显示环比变化百分比

#### 5.2.3 跳转交互

- 列表页点击人群名称 → 进入人群详情页
- 详情页点击"查看个体标签" → 新标签页打开个人数字健康空间
- 面包屑导航：人群健康空间 > 人群名称

## 六、数据字典

### 6.1 人群分组表

| 字段名 | 字段类型 | 长度 | 必填 | 说明 |
|--------|---------|------|------|------|
| id | varchar | 32 | 是 | 主键 |
| group_name | varchar | 50 | 是 | 人群名称 |
| group_rules | json | - | 是 | 分组规则（标签组合） |
| remark | varchar | 200 | 否 | 备注描述 |
| population_count | int | - | 是 | 人群数量 |
| status | tinyint | - | 是 | 状态（1已完成/2计算中/3已过期） |
| create_time | datetime | - | 是 | 创建时间 |
| update_time | datetime | - | 是 | 更新时间 |
| creator_id | varchar | 32 | 是 | 创建人ID |

### 6.2 分组规则JSON结构示例

```json
{
  "logic": "and",  // 逻辑关系：and/or
  "conditions": [
    {
      "type": "disease",  // 标签类型
      "label": "高血压",
      "value": "hypertension"
    },
    {
      "type": "age",
      "label": "60-70岁",
      "value": "60-70"
    },
    {
      "type": "gender",
      "label": "女性",
      "value": "female"
    }
  ]
}
```

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 人群计算性能不足 | 计算时间过长 | 采用异步计算+缓存机制 |

| 数据权限复杂 | 开发周期延长 | 优先实现基础权限，后续迭代 |
